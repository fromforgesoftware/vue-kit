## Design principles

- **Composable** — components follow the compound component pattern, exposing sub-components (e.g. `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuContent`)
- **Headless primitives** — built on Reka UI for accessible, unstyled behaviour
- **CVA variants** — all styling is driven by `class-variance-authority` variant definitions in `{component}.ts` files
- **Tailwind CSS** — utility-first styling with design tokens via CSS variables
