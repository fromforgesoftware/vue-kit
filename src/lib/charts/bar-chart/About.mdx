# Bar Chart

Vertical or horizontal bar chart with optional grouping, stacking, and rounded corners. Wraps Chart.js via vue-chartjs and exposes `role="img"` plus an accessible name so screen readers can announce what the chart represents.

## When to use

- Compare values across discrete categories.
- Show trends over an evenly spaced index axis (months, weeks, days).
- Stack contributions to a total (with `stacked`).

## When not to use

- For continuous data where trajectory matters — use [Line Chart](?path=/docs/charts-linechart--docs).
- For part-to-whole at a single point in time — use [Donut Chart](?path=/docs/charts-donutchart--docs).

## Composition

The `data` prop drives everything:

```ts
import { type BarChartData, DEFAULT_BAR_COLORS } from '@fromforgesoftware/vue-kit';

const data: BarChartData = {
	categories: ['Q1', 'Q2', 'Q3', 'Q4'],
	datasets: [
		{ label: 'Revenue', color: DEFAULT_BAR_COLORS[0], data: [12, 18, 22, 31] },
		{ label: 'Expenses', color: DEFAULT_BAR_COLORS[1], data: [9, 12, 14, 16] },
	],
};
```

`DEFAULT_BAR_COLORS` resolves to the `--color-chart-1` … `--color-chart-5` design tokens, so charts respect the active theme automatically. Pass any CSS colour string per dataset to override.

## Accessibility (WCAG 2.2 AA)

- **Role and name** (SC 1.1.1, 4.1.2): the wrapper has `role="img"` and `aria-label`. When `ariaLabel` is omitted, a name is auto-derived from the dataset labels.
- **Long description**: pass `ariaDescribedBy` pointing to a `<p id="chart-desc">` summarising the data trend for users who can't see the visualisation.
- **Colour contrast** (SC 1.4.11): `DEFAULT_BAR_COLORS` resolves to theme-token colours that meet ≥ 3 : 1 against the chart background. If you override colours, verify contrast manually.
- **Keyboard interaction**: Chart.js does not expose bar-level keyboard navigation. Pair the chart with a sibling table (visually hidden if needed) when keyboard inspection of values is required.
- **Tooltip**: tooltips are pointer-only by Chart.js default. Consumers who need keyboard tooltips should provide an alternative data table.

## Responsive behaviour

<Canvas of={BarChartStories.InteractiveResponsive} />

The chart fills its container width and uses the `height` prop for vertical sizing. Wrap in a `<div class="overflow-hidden">` when nesting in horizontally scrolling layouts to avoid resize loops.
