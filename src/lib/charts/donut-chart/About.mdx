# Donut Chart

Part-to-whole chart with optional centre content (totals, percentage, indicator). Wraps Chart.js and exposes `role="img"` with an accessible name. Three legend variants and a `minimal` mode without a legend cover most dashboard layouts.

## When to use

- Show distribution at a single point in time (job statuses, traffic sources).
- Surface a headline number inside a visual context (centre slot).

## When not to use

- For comparisons across time — use [Bar Chart](?path=/docs/charts-barchart--docs).
- For more than ~6 segments — small slices become unreadable; use a horizontal bar chart instead.

## Composition

```ts
import { type DonutSegment, DEFAULT_DONUT_COLORS } from '@fromforgesoftware/vue-kit';

const segments: DonutSegment[] = [
	{ value: 100, label: 'Active', color: DEFAULT_DONUT_COLORS[0] },
	{ value: 48, label: 'In review', color: DEFAULT_DONUT_COLORS[1] },
	{ value: 26, label: 'Finished', color: DEFAULT_DONUT_COLORS[2] },
];
```

`DEFAULT_DONUT_COLORS` resolves to `--color-chart-1` … `--color-chart-5` design tokens. Each segment carries its own `color` so consumers can paint segments by domain meaning (status colours, brand colours, etc.).

`DonutChartLegend` is exported separately and can be rendered standalone — useful when grouping multiple donuts under one shared legend.

Pass a `center` slot for richer content (delta indicator, sparkline, secondary metric). Centre content is `aria-hidden` because the segment values are already in the chart's `aria-label`.

## Accessibility (WCAG 2.2 AA)

- **Role and name** (SC 1.1.1, 4.1.2): the wrapper carries `role="img"` and `aria-label`. When `ariaLabel` is omitted, a name is auto-derived listing each segment label and value.
- **Centre content**: marked `aria-hidden` because the data is already announced via the chart's accessible name — avoids duplicate readout.
- **Long description**: pass `ariaDescribedBy` pointing to a paragraph summarising the distribution for screen readers.
- **Colour contrast**: `DEFAULT_DONUT_COLORS` resolves to theme-token colours. The 2 px borderless layout in `minimal` mode uses a 4 px gap (`spacing`) to separate segments without relying on contrast alone.
- **Keyboard interaction**: Chart.js does not expose segment-level keyboard navigation. Pair with a sibling table for keyboard inspection if needed.

## Responsive behaviour

<Canvas of={DonutChartStories.InteractiveResponsive} />

Use `legend-bottom` in narrow columns; `minimal` in tightly packed KPI grids. The chart canvas respects the explicit `size` prop — wrap it in a flex container if you want it to fill available space.
