# Color Area

Two-dimensional colour picker. By default the X axis controls hue (or saturation in HSB) and the Y axis controls saturation (or value). Built on Reka UI; pair with a hue [Color Slider](?path=/docs/color-color-slider--docs) for full HSB control.

## When to use

- Build a custom colour picker UI alongside hue/alpha sliders and a hex field.
- Let users explore a 2D colour space directly (saturation × value).

## When not to use

- For one-channel adjustments (hue alone, alpha alone) — use [Color Slider](?path=/docs/color-color-slider--docs).
- For predefined palettes — use [Color Swatch Picker](?path=/docs/color-color-swatch-picker--docs).

## Anatomy

<Canvas of={ColorAreaStories.Default} />

The component renders a Reka root that holds the gradient, a focusable thumb that meets `aria-roledescription="Color thumb"`, and a hidden form input when used inside a form.

## Color spaces

<Canvas of={ColorAreaStories.HSB} />

`hsl` (default) and `hsb` are supported. Choose the colour space that matches the rest of your picker — mixing `hsl` here with `hsb` on the slider produces unexpected drift on edge cases.

## Accessibility (WCAG 2.2 AA)

- **Pointer alternatives** (SC 2.5.7): every drag interaction has a keyboard equivalent — arrows, Page keys, Home / End — so the picker is fully usable without a pointing device.
- **Target size** (SC 2.5.8): the thumb's visible knob is 16 px, with an invisible expansion ring (`::before`) pushing the pointer hit-target to ≥ 24 × 24 px.
- **Name** (SC 4.1.2): the thumb auto-derives an `aria-label` from the axis channel names. The root accepts an `ariaLabel` prop for higher-level context (e.g. `"Brand colour picker"`).
- **Focus visible** (SC 2.4.7): the thumb shows a 4-px ring on `focus-visible`.
- **Disabled state**: when `disabled` is true the root carries `data-disabled` and the thumb is not focusable; opacity drops to 50 %.

## Responsive behaviour

<Canvas of={ColorAreaStories.InteractiveResponsive} />

Use `size="full"` when the picker should fill its parent — the inner area uses `size-full` with a `min-h-[160px]` floor so the gradient never collapses.

		</tr>
		<tr>
			<td>
				Pass an <code>ariaLabel</code> describing what the picker controls
			</td>
