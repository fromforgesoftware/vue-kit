# Color Slider

Single-channel colour slider built on Reka UI. Behaves like the standard [Slider](?path=/docs/form-slider--docs) but the track auto-paints a gradient that visualises the channel being edited.

## When to use

- Hue, saturation, lightness, brightness, alpha, or RGB channel editors inside a custom picker.
- Pair with a [Color Area](?path=/docs/color-color-area--docs) for full HSB control.

## When not to use

- For numeric ranges with no colour meaning — use [Slider](?path=/docs/form-slider--docs).
- For predefined palettes — use [Color Swatch Picker](?path=/docs/color-color-swatch-picker--docs).

## Accessibility (WCAG 2.2 AA)

- **Pointer alternatives** (SC 2.5.7): every drag interaction has a keyboard equivalent.
- **Target size** (SC 2.5.8): visible thumb stays compact for clean visuals; an invisible `::before` ring delivers ≥ 24 × 24 px hit area.
- **Name** (SC 4.1.2): the thumb auto-derives an `aria-label` from the channel (e.g. `"Hue"`, `"Saturation"`). Override at the root via `ariaLabel` for higher-level context.
- **Value** (SC 4.1.2): `aria-valuenow` and `aria-valuetext` are populated by Reka — `aria-valuetext` reads `"45"` for hue, `"60%"` for percent channels, etc.

## Responsive behaviour

<Canvas of={ColorSliderStories.InteractiveResponsive} />

Sliders collapse to fill their parent. Stack vertically inside a `grid gap-3` container in narrow popovers; switch to `orientation="vertical"` for tall sidebars.

			<th>Don't</th>

		<tr>
			<td>
				Pair each slider with a <code>&lt;Label&gt;</code> for clear identification
			</td>
			<td>
				Rely on the auto-derived <code>aria-label</code> when several sliders share a parent
			</td>
		</tr>
		<tr>
			<td>
				Match <code>colorSpace</code> across all sliders in a picker
			</td>
			<td>
				Mix HSL <code>lightness</code> with HSB <code>brightness</code> in the same picker
			</td>
