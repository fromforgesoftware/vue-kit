# Color Swatch

Display-only colour chip. Renders as `role="img"` with an accessible label automatically derived from the colour value, so screen readers announce "blue" rather than `#3b82f6`. For pickable swatches, use `ColorSwatchPicker` instead.

## When to use

- Show a colour value next to a label (palette legend, theme token table).
- Render an inline preview chip alongside a colour input.
- Annotate a series colour in a chart legend at the consumer.

## When not to use

- For user-selectable colour grids — use [Color Swatch Picker](?path=/docs/color-color-swatch-picker--docs).
- For interactive previews that open a picker — use [Color Picker](?path=/docs/color-color-picker--docs).

## Anatomy

<Canvas of={ColorSwatchStories.Default} />

A single inline element with an `aria-label` and `aria-roledescription="color swatch"`. The inset ring softens the edge against the page so near-white swatches stay legible.

## Accessibility (WCAG 2.2 AA)

- **Role** (SC 1.3.1): rendered as `role="img"` with `aria-roledescription="color swatch"`. Screen readers announce the colour name, not the hex.
- **Name** (SC 4.1.2): every swatch has an accessible name — derived from the colour or overridden via `label`.
- **Contrast** (SC 1.4.11): the inset ring guarantees ≥ 3 : 1 contrast against the page surface even for near-white swatches.
- **Target size** (SC 2.5.8): not interactive — does not need to meet the 24 × 24 px minimum. If a consumer wraps the swatch in a click handler, they must enforce target size at the wrapper.

## Responsive behaviour

<Canvas of={ColorSwatchStories.InteractiveResponsive} />

Swatches are inline-flex with `shrink-0` — they preserve their dimensions inside `flex-wrap` containers and never overflow horizontally.
