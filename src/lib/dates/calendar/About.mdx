# Calendar

Always-visible single-month date picker built on Reka UI's Calendar primitive. Use it inline; for a popover-triggered picker compose the [DatePicker](?path=/docs/dates-datepicker--docs) instead.

## When to use

- The user must always see the calendar (forms with prominent date selection, scheduling layouts).
- A specific day must be picked from a continuous range.
- Date constraints (min/max, unavailable rules) need direct visualisation.

## When not to use

- Popover-style date entry — use [DatePicker](?path=/docs/dates-datepicker--docs).
- Date _and_ time entry — use [DateField](?path=/docs/dates-datefield--docs) for keyboard segmented input.
- Range selection — use [RangeCalendar](?path=/docs/dates-rangecalendar--docs).

## Anatomy

<Canvas of={CalendarStories.Default} />

`CalendarRoot` wraps a header (`CalendarPrev`, `CalendarHeading` or month/year `<select>` controls, `CalendarNext`) and one or more `CalendarGrid` blocks containing day cells.

## States

<Canvas of={CalendarStories.WithMinMax} />

<Canvas of={CalendarStories.WithUnavailableDates} />

`disabled` blocks all interaction. `minValue`/`maxValue` softly disable individual cells. `isDateUnavailable` is for context-specific rules (holidays, blocked days) — those cells render struck-through.

## Composition

<Canvas of={CalendarStories.WithMonthYearSelect} />

<Canvas of={CalendarStories.TwoMonths} />

`monthYearSelect` swaps the heading text for two `<select>` controls — useful when the user picks dates far from today. `numberOfMonths={2}` shows two months side by side; the layout wraps to a column on narrow viewports.

## Accessibility (WCAG 2.2 AA)

- **1.4.3 Contrast**: foreground tokens against `bg-background` meet 4.5 : 1; selected cell uses primary tokens (verified ≥ 4.5 : 1).
- **2.1.1 Keyboard**: full grid navigation via Arrow / Home / End / PageUp / PageDown and Enter / Space activation.
- **2.4.7 Focus visible**: nav buttons and day cells render a 2 px ring at ≥ 3 : 1 contrast.
- **2.5.8 Target size**: every day cell is ≥ 24 × 24 px at all sizes; nav buttons are 24 / 28 / 32 px.
- **4.1.2 Name & state**: `CalendarRoot` renders `role="application"` with a labelled grid; selected cells expose `aria-selected="true"` and today carries `aria-current="date"` (Reka).

## Responsive behaviour

<Canvas of={CalendarStories.InteractiveResponsive} />

The root is `w-fit` — content drives width. `numberOfMonths={2}` wraps to a column on `< sm` breakpoints (`flex-col sm:flex-row`).

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Use <code>size="sm"</code> when the calendar appears in a sidebar or filter panel.
			</td>
			<td>
				Override <code>class</code> to shrink cells — the size variant keeps target sizes
				WCAG-compliant.
			</td>
		</tr>
		<tr>
			<td>
				Combine <code>minValue</code>/<code>maxValue</code> with <code>isDateUnavailable</code> for
				hard limits + soft rules.
			</td>
			<td>
				Block invalid dates only via <code>isDateDisabled</code> when the rule is a hard limit — use{' '}
				<code>maxValue</code>.
			</td>
		</tr>
		<tr>
			<td>
				Switch to <code>monthYearSelect</code> for date-of-birth and far-historic pickers.
			</td>
			<td>
				Force users to click prev/next dozens of times — give them <code>monthYearSelect</code>{' '}
				instead.
			</td>
		</tr>
		<tr>
			<td>
				Wrap in <code>w-fit</code> containers — Calendar already sizes itself.
			</td>
			<td>
				Stretch the Calendar to <code>w-full</code>; the grid won't reflow correctly.
			</td>
		</tr>
	</tbody>
</table>
