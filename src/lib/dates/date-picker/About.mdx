# DatePicker

Composite single-date picker — segmented date field or button trigger paired with a popover Calendar. Built on Reka UI's `DatePicker` primitives. Use it whenever the user needs both keyboard date entry and visual month browsing.

## When to use

- Forms requiring a single date with optional preset shortcuts.
- Filters or scheduling panels where the calendar opens on demand instead of staying inline.
- Date + time entry via `granularity="minute"` (segmented field variant).

## When not to use

- Always-visible calendar — use [Calendar](?path=/docs/dates-calendar--docs).
- Pure keyboard date entry without a popover — use [DateField](?path=/docs/dates-datefield--docs).
- Date _range_ selection — use [DateRangePicker](?path=/docs/dates-daterangepicker--docs).

## Anatomy

<Canvas of={DatePickerStories.Default} />

`DatePickerRoot` wraps either a segmented `DatePickerField` (with calendar-icon `DatePickerTrigger`) or a single `DatePickerTrigger` button. The popover content (`DatePickerContent`) hosts a `DatePickerCalendar` and an optional preset sidebar.

## Composition

<Canvas of={DatePickerStories.WithMonthYearSelect} />

<Canvas of={DatePickerStories.TwoMonths} />

<Canvas of={DatePickerStories.WithDayPresets} />

`monthYearSelect` swaps the heading for two `<select>` controls. `numberOfMonths={2}` shows two months side by side. `presets` adds a sidebar of one-click shortcuts (e.g. Today, Yesterday, Start of Week) — selecting one updates the model and highlights the active preset.

`<DatePickerPresets>` is exported separately so consumers can build custom preset components.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair every picker with `<Label for="id">` and pass `placeholder` for screen-reader context on the button variant.
- **3.3.1 Error identification**: use the `error` prop to set `aria-invalid="true"`; reference helper text via `describedBy`.
- **2.4.7 Focus visible**: trigger renders a 2 px ring on focus; nav buttons and day cells render their own rings inside the popover.
- **2.5.8 Target size**: every size's trigger and day cells meet ≥ 24 × 24 px.
- **4.1.2 Name & state**: the popover dialog announces correctly; selected cells expose `aria-selected="true"` and today carries `aria-current="date"` (Reka).

## Responsive behaviour

<Canvas of={DatePickerStories.InteractiveResponsive} />

The trigger is `w-fit` — it shrinks to its content. The popover is portaled to `document.body` and positioned by Reka's collision-aware logic. Two-month layouts stack to a column at narrow widths.
