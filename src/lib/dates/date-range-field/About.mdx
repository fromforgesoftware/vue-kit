# DateRangeField

Segmented date-range input — start and end dates each have their own focusable segments separated by an en-dash. Built on Reka UI's `DateRangeField` primitive.

## When to use

- Fast keyboard date-range entry inside a form (filters, reports, exports).
- Free-typing a known range — segments accept digits and snap on overflow.
- Range with optional time portion via `granularity`.

## When not to use

- Visual range browsing — use [RangeCalendar](?path=/docs/dates-rangecalendar--docs) or [DateRangePicker](?path=/docs/dates-daterangepicker--docs).
- Single date entry — use [DateField](?path=/docs/dates-datefield--docs).

## Anatomy

<Canvas of={DateRangeFieldStories.Default} />

`DateRangeFieldRoot` wraps two segment groups (start, end) and an en-dash separator. Each data segment is `role="spinbutton"` with min / max bounds.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair every field with `<Label>`.
- **3.3.1 Error identification**: pass `error` to set `aria-invalid="true"` and use `describedBy` to reference helper text.
- **2.4.7 Focus visible**: focused segments highlight with `bg-accent`; the wrapper renders a 2 px ring.
- **2.5.8 Target size**: every size's wrapper meets ≥ 24 × 24 px.
- **4.1.2 Name & state**: each segment exposes `role="spinbutton"`. The end-segment group is announced separately from start.

## Responsive behaviour

<Canvas of={DateRangeFieldStories.InteractiveResponsive} />

The root is `w-fit`. On narrow viewports (`< sm`) consider stacking the field below its label.
