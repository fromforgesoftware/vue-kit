# DayPeriodRangeToggle

A **renderless** behaviour helper that drives the AM/PM segments on both sides of a [`TimeRangeField`](?path=/docs/dates-timerangefield--docs) without going through Reka's built-in keydown handler.

It exists to fix a subtle bug: Reka's default `dayPeriod` keydown handler applies a ┬▒12-hour shift to a _null_ `hour` segment, which silently changes the displayed clock when the user just wants to flip AM/PM on an empty side of the range. `DayPeriodRangeToggle` writes `dayPeriod` directly onto `segmentValues[side]` and only adjusts `hour` when it is already set.

## When to use

- Internally, by `TimeRangeField`, to wire up its start / end dayPeriod segments.
- In any custom field that uses `TimeRangeFieldRoot` and needs reliable AM/PM toggling per range side ŌĆö render the helper inside the root's slot and grab a template ref.

## When not to use

- For the single-time version (`TimeFieldRoot`) ŌĆö use [`DayPeriodToggle`](?path=/docs/dates-dayperiodtoggle--docs) which targets a single segment with no `side` argument.
- As a visual control ŌĆö this component renders only its slot, no DOM, no styling.

## Anatomy

<Canvas of={DayPeriodRangeToggleStories.Default} />

The component renders only the default `<slot />`. It must be placed inside a `TimeRangeFieldRoot` slot scope so `injectTimeRangeFieldRootContext()` resolves.

```vue
<TimeRangeFieldRoot>
  <template #default="{ segments }">
    <DayPeriodRangeToggle ref="dayPeriodToggleRef" />
    <!-- ŌĆ” start + end segments ŌĆ” -->
  </template>
</TimeRangeFieldRoot>

<script setup lang="ts">
import { ref } from 'vue';
const dayPeriodToggleRef = ref<InstanceType<typeof DayPeriodRangeToggle>>();

function flipStart() {
	dayPeriodToggleRef.value?.toggle('start');
}
</script>
```

Both methods preserve the current `hour` value when it is `null` and only adjust it (┬▒12) when an explicit hour is already set. Sides are always independent ŌĆö toggling `start` never touches `end` and vice versa.

## Accessibility (WCAG 2.2 AA)

- **No DOM contribution**: the component renders only its slot, so it adds nothing to the accessibility tree on its own.
- **Naming and state are the host's responsibility**: each dayPeriod segment in `TimeRangeField` is a `role="spinbutton"` with `aria-valuetext` reflecting AM / PM, and carries `data-range-side="start" | "end"` for assistive-tech-aware pickers.
- **2.1.1 Keyboard**: pairs with `TimeRangeField`'s segment-level keyboard model ŌĆö Arrow / A / P / click all work for whichever side has focus.
- **Side independence**: toggling one side must not affect the other; the `InteractiveSidesAreIndependent` test asserts this.
- **Hour preservation**: critical correctness rule ŌĆö toggling AM/PM on a side with `hour === null` must not silently shift the displayed hour. The `InteractiveHourPreserved` test asserts this.
