# DayPeriodToggle

A **renderless** behaviour helper that drives the AM/PM segment of a [`TimeField`](?path=/docs/dates-timefield--docs) without going through Reka's built-in keydown handler.

It exists to fix a subtle bug: Reka's default `dayPeriod` keydown handler applies a ┬▒12-hour shift to a _null_ `hour` segment, which silently changes the displayed clock when the user just wants to flip AM/PM on an empty field. `DayPeriodToggle` writes `dayPeriod` directly onto `segmentValues` and only adjusts `hour` when it is already set.

## When to use

- Internally, by `TimeField`, to wire up its dayPeriod segment.
- In any custom field that uses `TimeFieldRoot` and needs reliable AM/PM toggling ŌĆö render the helper inside the root's slot and grab a template ref.

## When not to use

- For the range version (`TimeRangeFieldRoot`) ŌĆö use [`DayPeriodRangeToggle`](?path=/docs/dates-dayperiodrangetoggle--docs) which targets `start` / `end` segments.
- As a visual control ŌĆö this component renders only its slot, no DOM, no styling.

## Anatomy

<Canvas of={DayPeriodToggleStories.Default} />

The component renders only the default `<slot />`. It must be placed inside a `TimeFieldRoot` slot scope so `injectTimeFieldRootContext()` resolves.

```vue
<TimeFieldRoot>
  <template #default="{ segments }">
    <DayPeriodToggle ref="dayPeriodToggleRef" />
    <!-- ŌĆ” segments ŌĆ” -->
  </template>
</TimeFieldRoot>

<script setup lang="ts">
import { ref } from 'vue';
const dayPeriodToggleRef = ref<InstanceType<typeof DayPeriodToggle>>();

function flip() {
	dayPeriodToggleRef.value?.toggle();
}
</script>
```

## Accessibility (WCAG 2.2 AA)

- **No DOM contribution**: the component renders only its slot, so it adds nothing to the accessibility tree on its own.
- **Naming and state are the host's responsibility**: the dayPeriod segment in `TimeField` exposes `role="spinbutton"` with `aria-valuetext` reflecting AM / PM.
- **2.1.1 Keyboard**: pairs with `TimeField`'s segment-level keyboard model ŌĆö Arrow / A / P / click all work.
- **Hour preservation**: critical correctness rule ŌĆö toggling AM/PM on a field with `hour === null` must not silently shift the displayed hour. The `InteractiveHourPreserved` test asserts this.
