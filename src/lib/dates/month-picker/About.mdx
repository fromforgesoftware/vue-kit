# MonthPicker

Composite month picker — button trigger that opens a popover with a 4 × 3 grid of month cells. Built on Reka UI's `MonthPicker` primitives.

## When to use

- Reports and analytics where the smallest meaningful unit is a calendar month.
- Filters where the user picks a single month at a time (e.g. timesheets, payroll).
- Inline scheduling layouts via the `inline` variant when the month grid should always be visible.

## When not to use

- Specific day selection — use [DatePicker](?path=/docs/dates-datepicker--docs).
- Month _range_ selection — use [MonthRangePicker](?path=/docs/dates-monthrangepicker--docs).
- Year-only granularity — use [YearPicker](?path=/docs/dates-yearpicker--docs).

## Anatomy

<Canvas of={MonthPickerStories.Default} />

`MonthPickerRoot` wraps a `MonthPickerHeader` (year navigation + heading) and a `MonthPickerGrid` of 12 month cells. In the `button` variant the root lives inside a popover triggered by an outline button.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair every picker with `<Label for="id">`.
- **3.3.1 Error identification**: `error` + `describedBy` over colour alone.
- **2.4.7 Focus visible**: 2 px ring on trigger, nav buttons, and cell triggers.
- **2.5.8 Target size**: trigger and cells meet ≥ 24 × 24 px at every size.
- **4.1.2 Name & state**: selected cell exposes `data-selected`; nav buttons carry `aria-label`s.

## Responsive behaviour

<Canvas of={MonthPickerStories.InteractiveResponsive} />

The trigger is `w-fit`. The popover is portaled to `document.body` and Reka positions it collision-aware.
