# RangeCalendar

Always-visible date-range picker built on Reka UI's RangeCalendar primitive. Selecting a start day then an end day creates a range. For a popover-triggered version compose [DateRangePicker](?path=/docs/dates-daterangepicker--docs).

## When to use

- The user must always see the range calendar (filters, scheduling layouts, dashboards).
- A start _and_ end day must be picked from a continuous range.
- Preset shortcuts (Today, This Week, Last Month) help users select common ranges fast.

## When not to use

- Popover-triggered range entry — use [DateRangePicker](?path=/docs/dates-daterangepicker--docs).
- Single date selection — use [Calendar](?path=/docs/dates-calendar--docs).
- Time-of-day or date-time entry — combine with [DateField](?path=/docs/dates-datefield--docs) / [TimeField](?path=/docs/dates-timefield--docs).

## Anatomy

<Canvas of={RangeCalendarStories.Default} />

`RangeCalendarRoot` wraps a header (`RangeCalendarPrev`, `RangeCalendarHeading` or month/year `<select>`s, `RangeCalendarNext`) and one or more `RangeCalendarGrid` blocks. Cells inside the selected range carry `data-selected`; bookends carry `data-selection-start` / `data-selection-end`.

## Accessibility (WCAG 2.2 AA)

- **1.4.3 Contrast**: range bookends use primary tokens (≥ 4.5 : 1 contrast); in-range cells use accent tokens.
- **2.1.1 Keyboard**: full grid navigation; in-flight selection tracked via Reka's `data-highlighted` state.
- **2.4.7 Focus visible**: nav buttons and day cells render a 2 px ring at ≥ 3 : 1 contrast.
- **2.5.8 Target size**: every day cell is ≥ 24 × 24 px at all sizes.
- **4.1.2 Name & state**: `RangeCalendarRoot` exposes a labelled grid; selected cells carry `aria-selected="true"`, today carries `aria-current="date"` (Reka).

## Responsive behaviour

<Canvas of={RangeCalendarStories.InteractiveResponsive} />

The root is `w-fit`. Multi-month layouts wrap to a column on `< sm` viewports.

		</tr>
		<tr>
			<td>
				Use <code>numberOfMonths={2}</code> when ranges typically cross month boundaries.

			<td>
				Stretch the calendar to fill wide layouts — keep it <code>w-fit</code>.
			</td>

		<tr>
			<td>

				WCAG-compliant.
