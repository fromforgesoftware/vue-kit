# TimeRangeField

Segmented time-range input — start and end times each have their own focusable hour / minute (and optional second + AM/PM) segments. Built on Reka UI's `TimeRangeField` primitive.

## When to use

- Fast keyboard time-range entry (shift hours, lunch break, meeting slots).
- Free-typing a known range — segments accept digits and snap on overflow.

## When not to use

- Single time entry — use [TimeField](?path=/docs/dates-timefield--docs).
- Date-range entry — use [DateRangeField](?path=/docs/dates-daterangefield--docs).

## Anatomy

<Canvas of={TimeRangeFieldStories.Default} />

`TimeRangeFieldRoot` wraps two segment groups (start, end) with hour / minute pairs and an optional AM/PM toggle. They're separated by an en-dash.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair every field with `<Label>`.
- **3.3.1 Error identification**: pass `error` and `describedBy`.
- **2.4.7 Focus visible**: focused segments highlight with `bg-accent`; the wrapper renders a 2 px ring.
- **2.5.8 Target size**: every size's wrapper meets ≥ 24 × 24 px.
- **4.1.2 Name & state**: each segment exposes `role="spinbutton"` with min / max bounds.

## Responsive behaviour

<Canvas of={TimeRangeFieldStories.InteractiveResponsive} />

The root is `w-fit`. On narrow viewports (`< sm`) consider stacking start and end as separate `TimeField`s instead.
