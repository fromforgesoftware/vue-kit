# Autocomplete

Searchable input with a filtered dropdown. Built on Reka UI's `Combobox` primitive — typing filters items, arrow keys navigate, Enter selects.

## When to use

- 50+ options where typing-as-filter is faster than scrolling.
- Server-side search (use `loading` + `@search` to show a spinner during fetches).
- Single-value selection from a long catalogue (countries, employees, products).

## When not to use

- Free-form text — use [Input](?path=/docs/form-input--docs).
- Short fixed list (≤ 10) — use [Select](?path=/docs/form-select--docs) for less typing overhead.
- Multi-select with chips — use [Select](?path=/docs/form-select--docs) with `multiple`.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair the input with `<Label for>` matching the `id` you set, or pass `aria-label`.
- **3.3.1 Error identification**: `error` sets `aria-invalid="true"`; reference helper text via `describedBy`.
- **2.4.7 Focus visible**: anchor draws a 2 px ring on focus-within.
- **2.5.8 Target size**: anchor and items meet the 24 × 24 px minimum at every size.
- **4.1.2 Name, role, value**: built on Reka UI primitives that emit the correct combobox / listbox / option semantics with `aria-expanded` / `aria-controls`. The clear and trigger buttons are icon-only — accessible names come from `clearLabel` (default `"Clear"`) and `triggerLabel` (default `"Open options"`).

## Responsive behaviour

<Canvas of={AutocompleteStories.InteractiveResponsive} />

The anchor is `w-full` and shrinks to its parent. Wrap in a sized container (`w-72`, `max-w-sm`) so very long values don't stretch wide layouts.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Pair every input with <code>&lt;Label for&gt;</code> or <code>aria-label</code>.
			</td>
			<td>Rely on placeholder as the only label.</td>
		</tr>
		<tr>
			<td>
				Use <code>error</code> + <code>describedBy</code> for validation.
			</td>
			<td>
				Toggle border colour with classes — it skips <code>aria-invalid</code>.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>@search</code> for server-side filtering with <code>loading</code>.
			</td>
			<td>
				Block UI while fetching — set <code>loading</code> instead.
			</td>
		</tr>
		<tr>
			<td>Reach for Select when there are fewer than ten options.</td>
			<td>Force users to type when scrolling is faster.</td>
		</tr>
	</tbody>
</table>
