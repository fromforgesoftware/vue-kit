# Checkbox

Boolean form control with optional indeterminate (mixed) state. Built on Reka UI's `CheckboxRoot` primitive.

## When to use

- Toggle a single boolean (accept terms, enable feature).
- Multi-select inside a list — one checkbox per option, no max.
- Parent-of-mixed-children rows — set `checked="indeterminate"`.

## When not to use

- Mutually exclusive choice — use [Radio](?path=/docs/form-radio--docs).
- Settings that take immediate effect — use [Switch](?path=/docs/form-switch--docs).

## Anatomy

<Canvas of={CheckboxStories.Default} />

A square painted control with a tick (checked) or dash (indeterminate) inside, plus an invisible padding ring that expands the hit area to ≥ 24 × 24 px.

## Variants

<Canvas of={CheckboxStories.Variants} />

Three states: unchecked (`false`), checked (`true`), and indeterminate.

## Composition

<Canvas of={CheckboxStories.WithLabel} />

Pair Checkbox with Label using `for` matching the checkbox's `id`. This:

- Makes clicking the label toggle the box.
- Gives the box a programmatic accessible name.
- Lets `getByLabelText` find the checkbox in tests.

<Canvas of={CheckboxStories.Group} />

For groups, render one Checkbox per option inside its own Label.

## Accessibility (WCAG 2.2 AA)

- **2.5.8 Target size**: visual square is < 24 px to keep dense forms tidy, but the `::before` ring expands the click target to ≥ 24 × 24 px. Composing with a Label inside a `flex` row gives a row-wide hit area that always meets the minimum.
- **4.1.2 Name, role, value**: Reka exposes `role="checkbox"` and `aria-checked` (`true`, `false`, or `mixed` for indeterminate). Always pair with a Label — never rely on adjacent text alone.
- **3.3.1 Error identification**: pass `error` to set `aria-invalid="true"` and use `describedBy` for an error message id.
- **2.1.1 Keyboard**: Space toggles the value. Tab moves focus.
- **2.4.7 Focus visible**: focus draws a 2 px ring + 2 px offset against the surrounding surface.
- **1.4.11 Non-text contrast**: the checked / indeterminate state uses `bg-primary` which meets 3 : 1 against the background.

## Responsive behaviour

<Canvas of={CheckboxStories.InteractiveResponsive} />

Checkboxes don't change layout across breakpoints. The hit area is constant at every viewport.

				Use <code>indeterminate</code> for parent rows where some children are checked.
			</td>
			<td>
				Use <code>indeterminate</code> as a "third" valid value — it should resolve on next click.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>error</code> + a referenced helper text id for validation.
			</td>
			<td>Hide the error message and only colour the box red.</td>
		</tr>
		<tr>
			<td>
				Set <code>required</code> for fields that block form submission (e.g. Accept terms).
			</td>

				Add a visual <code>*</code> in the label slot — let the Label component handle it.
