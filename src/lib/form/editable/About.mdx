# Editable

An inline-editable text field. Click or focus the preview to enter edit mode; press Enter to save and Escape to cancel. Built on Reka `Editable` — focus management and keyboard handling are taken care of for you.

## When to use

- Light-touch rename of a list item, board column, or breadcrumb crumb where opening a form would feel heavy.
- Editing a single text value in place — title, label, name.

## When not to use

- Multi-field forms — use a Dialog or a real form layout.
- Validation-heavy or required input — use [FormField](?path=/docs/form-formfield--docs) + [Input](?path=/docs/form-input--docs) so you can show error messages.
- Numeric values with steppers — use [NumberField](?path=/docs/form-numberfield--docs).

## Sizes

<Canvas of={EditableStories.Sizes} />

`size` on the parent flows down to Preview / Input / Triggers via Vue provide/inject — set it once. Heights track the Input ramp (`sm` 28 px, `default` 32 px, `lg` 40 px) so the preview ↔ input swap doesn't shift the row.

`disabled` blocks activation and lowers opacity. `readonly` shows the preview but never enters edit mode. Empty values render the placeholder in muted text — the preview keeps its 24×24 hit area regardless.

## Explicit triggers

<Canvas of={EditableStories.WithExplicitTriggers} />

For an Edit / Save / Cancel pattern set both `activationMode="none"` and `submitMode="none"`, then drop in `EditableEditTrigger`, `EditableSubmitTrigger`, `EditableCancelTrigger`. Every trigger is ≥ 24×24 CSS px regardless of size.

## Accessibility (WCAG 2.2 AA)

- **2.4.3 Focus order**: focus enters the preview in tab order; activation moves focus to the input; saving / cancelling returns focus to the preview.
- **2.4.7 Focus visible**: both preview and input draw a ring at ≥ 3 : 1 contrast.
- **2.5.8 Target size**: every size's preview and trigger meets ≥ 24×24 CSS px.
- **3.3.5 Help / 3.3.1 Error identification**: Editable does not surface errors itself — pair with a [FormField](?path=/docs/form-formfield--docs) parent for validation messaging.
- **4.1.2 Name, role, value**: pass `aria-label` (or text content) on each trigger so icon-only buttons announce their action.

## Responsive behaviour

<Canvas of={EditableStories.InteractiveResponsive} />

EditableArea is `inline-flex w-full min-w-0` so the preview and input shrink to fit narrow containers. Long values truncate inside the preview while editing remains scrollable inside the input.

## Do / Don't

		<tr>

			<td>

				Use the default <code>blur</code> mode in a row with hover-revealed actions — focus may
				shift accidentally.
			</td>
		</tr>
		<tr>
			<td>Use Editable for low-stakes rename in place.</td>
			<td>Use Editable for required validation — pair with FormField + Input.</td>
		</tr>
		<tr>
			<td>
				Pass <code>aria-label</code> on icon-only triggers.
			</td>
			<td>Rely on icon alone — assistive tech needs a name.</td>
		</tr>
	</tbody>
