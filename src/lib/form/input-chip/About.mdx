# InputChip

Multi-value chip input. Type a value and press <kbd>Enter</kbd> (or comma) to add it as a chip; press <kbd>Backspace</kbd> on an empty input to remove the last chip.

## When to use

- Free-form lists of strings: tags, labels, email recipients, skills, keywords.
- Where each entry should be visually separable and individually removable.

## When not to use

- Selecting from a known set of options — use [Select](?path=/docs/form-select--docs) (multiple) or [Autocomplete](?path=/docs/form-autocomplete--docs).
- A single value — use [Input](?path=/docs/form-input--docs).
- Read-only display of tags — use [Badge](?path=/docs/general-badge--docs).

## Anatomy

<Canvas of={InputChipStories.Default} />

A wrapper hosts the existing chips, a free-form text input, and an optional clear-all button. Each chip carries a remove button. The wrapper auto-grows in height as chips wrap.

## Chip variants

<Canvas of={InputChipStories.ChipVariants} />

`chipVariant` accepts a subset of Badge variants: `default`, `secondary`, `destructive`, `outline`, `soft-destructive`. Pick the variant that matches the kind of value being collected — `secondary` for neutral tags, `destructive` for blocked / blacklisted entries, `outline` for low-emphasis lists.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: Pair with a `<Label>` describing the collection ("Tags", "Recipients").
- **3.3.1 Error identification**: Use `error` + `describedBy` to surface validation. Both wrapper and input receive `aria-invalid="true"`.
- **2.5.8 Target size**: Per-chip remove buttons are square ≥ 24 × 24 px at every size.
- **2.4.7 Focus visible**: Wrapper draws a ring on focus-within; remove buttons draw their own ring.
- **iOS no-zoom**: input uses `text-sm` minimum (12 px) — prevents iOS Safari auto-zoom on focus.
- **Bounded entry**: `maxChips` disables the input once the limit is reached.

## Responsive behaviour

<Canvas of={InputChipStories.InteractiveResponsive} />

The wrapper is `w-full` and chips wrap onto new rows automatically. For mobile-first layouts, prefer shorter chip text (≤ 16 chars).
