# Input

Single-line text field. Wraps a native `<input>` so the caret area covers the full surface — even when leading/trailing icon affixes are present.

## When to use

- Free-form short text entry: name, email, phone, search, currency.
- One-line values inside a form, alongside a Label and optional helper/error text.

## When not to use

- Multi-line text — use [Textarea](?path=/docs/form-textarea--docs).
- Choice from a fixed list — use [Select](?path=/docs/form-select--docs) or [Radio](?path=/docs/form-radio--docs).
- Boolean — use [Checkbox](?path=/docs/form-checkbox--docs) or [Switch](?path=/docs/form-switch--docs).
- Numeric values with stepper / units — use [NumberField](?path=/docs/form-numberfield--docs).

## Anatomy

<Canvas of={InputStories.Default} />

The component renders a wrapper `<div>` (border, ring, slots) and a borderless native `<input>` inside. Optional `leading` / `trailing` slots render fixed-width affixes that share the wrapper's focus and error states.

Compose [Label](?path=/docs/form-label--docs) with `for` matching the input's `id`. This is the only way to give the field an accessible name — placeholder text is _not_ a label.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: every input must have a programmatic label — pair with `<Label for="...">` (or `aria-label` for icon-only search bars).
- **3.3.1 Error identification**: pass `error` to set `aria-invalid="true"` and use `describedBy` to point to a helper / error message id.
- **3.3.2 Labels or instructions**: helper text describing format expectations belongs in a `<p>` referenced via `aria-describedby`, not in `placeholder`.
- **2.4.7 Focus visible**: focus draws a 2 px ring at ≥ 3 : 1 contrast.
- **2.5.8 Target size**: every size's wrapper meets the 24 × 24 px minimum.
- **1.4.4 Resize text**: text uses `text-sm`; layouts remain usable at 200 % zoom.
- **iOS no-zoom**: `text-sm` (12 px Inter) is the minimum size that prevents iOS Safari from zooming the viewport on focus.

## Responsive behaviour

<Canvas of={InputStories.InteractiveResponsive} />

The wrapper is `w-full` and shrinks to its parent. Place inputs inside a max-width container (typically `w-72` or `max-w-sm`) so very long entries don't dominate wide layouts.
