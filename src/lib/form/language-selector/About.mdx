# LanguageSelector

Composite locale picker with two visual variants. Resolves bare base codes (e.g. `en`) to their first regional variant (e.g. `en_GB`) automatically.

## When to use

- Letting the user choose a locale in nav, footer, login screens, or settings.
- The list of locales comes from the backend as a string array.

## When not to use

- Selecting a single value from a non-locale list — use [Select](?path=/docs/form-select--docs) directly.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: the inline trigger sets `aria-label="Select language"` by default — override with the `ariaLabel` prop if your context calls for a different word.
- **2.4.7 Focus visible**: trigger draws a 2 px ring with offset.
- **2.5.8 Target size**: every trigger size meets the 24 × 24 px minimum.
- **4.1.2 Name, role, value**: built on Reka UI primitives that emit the correct combobox / menu / radio semantics.

## Responsive behaviour

<Canvas of={LanguageSelectorStories.InteractiveResponsive} />

The inline trigger is fixed-width to its content and never overflows. The select variant is `w-56` by default — wrap in a sized container if you need it narrower.

		</tr>
		<tr>
			<td>
				Pass the full <code>locales</code> array as supplied by your backend.
			</td>

			</td>
