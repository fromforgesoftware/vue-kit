# Listbox

Visible, keyboard-navigable list of selectable options. Built on Reka UI's `Listbox` primitive. Inline (not portaled) — the items are always on screen.

## When to use

- The user benefits from seeing all (or most) options at once.
- Option count is roughly 5–30 — small enough to scan, large enough to need a sized scroll container.
- Multi-select where chips would clutter the trigger.

## When not to use

- Saving vertical space — use [Select](?path=/docs/form-select--docs) (collapses into a trigger).
- Type-as-filter long catalogues — use [Autocomplete](?path=/docs/form-autocomplete--docs).
- Small (≤ 5) fixed list of toggles — use [Checkbox](?path=/docs/form-checkbox--docs) or [Radio](?path=/docs/form-radio--docs).

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair the listbox with a visible `<Label for>` matching the `id` you set, or wrap it in a labelled fieldset. When neither is practical, pass `ariaLabel` for an invisible accessible name.
- **3.3.1 Error identification**: `error` sets `aria-invalid="true"`; reference helper text via `describedBy`.
- **2.4.7 Focus visible**: focus draws a 2 px ring on the active item.
- **2.5.8 Target size**: every item meets the 24 × 24 px minimum.
- **4.1.2 Name, role, value**: built on Reka UI primitives that emit the correct ARIA listbox / option semantics with `aria-selected` reflecting state.

## Responsive behaviour

<Canvas of={ListboxStories.InteractiveResponsive} />

The root is `w-full` and shrinks to its parent. Wrap in a sized container so very long item labels don't stretch wide layouts.

## Do / Don't

	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>

	<tbody>
		<tr>
			<td>
				Pair the listbox with a <code>&lt;Label for&gt;</code>.
			</td>
			<td>Rely on placement alone for labelling.</td>
		</tr>
		<tr>
			<td>
				Use <code>multiple</code> for multi-select.
			</td>
			<td>Render two listboxes when the answer is a set.</td>
		</tr>
		<tr>
			<td>
				Use the <code>error</code> prop and a referenced helper text id.
			</td>
			<td>
				Override border colour with classes — it skips <code>aria-invalid</code>.
			</td>
		</tr>
		<tr>
			<td>Pick a size that fits roughly 5–10 items without scrolling.</td>
			<td>
				Set <code>max-height</code> via <code>class</code> — propose a new size variant.
			</td>
		</tr>
	</tbody>
