# NumberField

Numeric input with inline +/- buttons. Built on Reka NumberField. Supports min/max/step bounds, `Intl.NumberFormat` display, and Home/End shortcuts to jump to bounds.

## When to use

- Capturing a numeric quantity, count, currency, or unit value where stepping by a fixed increment is useful (qty, capacity, amount).
- Forms where users may prefer button-based adjustment over typing (touch, accessibility).

## When not to use

- Free-form numeric strings (phone, postcode) — use [Input](?path=/docs/form-input--docs) with `type="tel"`.
- A range slider over a continuous numeric range — use [Slider](?path=/docs/form-slider--docs).
- A choice from discrete numeric options — use [Select](?path=/docs/form-select--docs) or [Radio](?path=/docs/form-radio--docs).

## Anatomy

<Canvas of={NumberFieldStories.Default} />

The component is compound: `<NumberField>` (root) wraps `<NumberFieldDecrement>`, `<NumberFieldInput>`, and `<NumberFieldIncrement>`. The buttons share the input's borders so all three render as a single visual segment with focus rings rising above siblings via z-index.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: Pair the input with a `<Label for="...">` (place the `id` on `<NumberFieldInput>`). The +/- buttons are icon-only — `<NumberFieldIncrement>` and `<NumberFieldDecrement>` accept an `ariaLabel` prop (defaults to `"Increase value"` / `"Decrease value"`); their inner icons are marked `aria-hidden="true"`.
- **3.3.1 Error identification**: Use the `error` prop to set `aria-invalid="true"` and `describedBy` to point to a message id.
- **2.5.8 Target size**: +/- buttons are ≥ 24×24 at every size.
- **2.4.7 Focus visible**: input and buttons each draw their own focus ring; focused element rises via z-index so the ring is fully visible.
- **Keyboard**: Arrow keys, Home, End, PageUp, PageDown all supported (Reka primitive).
- **iOS no-zoom**: input uses `text-sm` (12 px) — minimum that prevents auto-zoom on focus.

## Responsive behaviour

<Canvas of={NumberFieldStories.InteractiveResponsive} />

The root is `inline-flex` so width is driven by the input width class (`w-36`, `w-40`, etc.). Place inside a max-width container so very long-formatted values don't dominate.

		<tr>
			<td>
				Set <code>min</code> / <code>max</code> for bounded ranges.
			</td>
			<td>Allow values that the calling system will reject — bound them client-side.</td>
		</tr>
		<tr>
			<td>
				Use <code>formatOptions</code> for currency / percentage display.
			</td>
			<td>Manually format inside the input — Reka rounds and parses for you.</td>
		</tr>
