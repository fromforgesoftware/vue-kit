# Radio

Mutually exclusive single-choice control. Built on Reka UI's `RadioGroupRoot` + `RadioGroupItem` primitives. Compose a `RadioGroup` with one `RadioGroupItem` per option and pair each item with a `Label`.

## When to use

- Pick one option from a small, fixed set (2–7 options).
- Show all options inline so the user can compare.

## When not to use

- Single boolean (yes/no, on/off) — use [Switch](?path=/docs/form-switch--docs) or [Checkbox](?path=/docs/form-checkbox--docs).
- More than ~7 options — use [Select](?path=/docs/form-select--docs).
- Multi-select — use [Checkbox](?path=/docs/form-checkbox--docs).

## Anatomy

<Canvas of={RadioStories.Default} />

A `RadioGroup` lays out one or more `RadioGroupItem`s along a vertical or horizontal axis. Each item is a circular button with a centred dot when selected. An invisible padding ring expands the click target to ≥ 24 × 24 px.

## Composition

`RadioGroup` is the container; `RadioGroupItem` is a single option. Items inherit `size` and `error` from the group via Vue provide/inject, but can override per item. Always pair each item with a `<Label for>` matching the item's `id`.

```vue
<RadioGroup v-model="plan">
  <Label for="plan-free" class="flex items-center gap-2 cursor-pointer">
    <RadioGroupItem id="plan-free" value="free" />
    Free
  </Label>
  <Label for="plan-pro" class="flex items-center gap-2 cursor-pointer">
    <RadioGroupItem id="plan-pro" value="pro" />
    Pro
  </Label>
</RadioGroup>
```

## Accessibility (WCAG 2.2 AA)

- **2.5.8 Target size**: visible circle is < 24 px to keep dense forms tidy, but the `::before` ring expands the click target to ≥ 24 × 24 px. Composing each item inside a Label inside a `flex` row gives a row-wide hit area that always meets the minimum.
- **4.1.2 Name, role, value**: Reka exposes `role="radiogroup"` on the group and `role="radio"` on each item. Always pair each item with a Label — the Label is the accessible name.
- **3.3.1 Error identification**: pass `error` on the group to set `aria-invalid="true"` on the group root and propagate to all items. Use `describedBy` on the group for the error message id.
- **2.1.1 Keyboard**: arrow keys move selection within the group; Tab moves focus in / out. Disabled items are skipped during arrow nav.
- **2.4.7 Focus visible**: each item draws a 2 px ring + 2 px offset against the surrounding surface.
- **1.4.11 Non-text contrast**: the selected item's border + dot use `text-primary` against `bg-background` — meets 3 : 1 against the surrounding surface.

## Responsive behaviour

<Canvas of={RadioStories.InteractiveResponsive} />

Radios don't change layout across breakpoints. The hit area is constant at every viewport. Long labels wrap inside the Label.

		</tr>

			<td>
				Use <code>error</code> + a referenced helper text id for validation.
			</td>
			<td>Hide the error message and only colour borders red.</td>
		</tr>
		<tr>
			<td>
				Pre-select a sensible default with <code>defaultValue</code> when there's a clear best
