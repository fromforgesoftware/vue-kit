# Select

Single- or multi-select dropdown built on Reka UI's `Select` primitive. The content portals to `document.body`, so it always escapes ancestors with `overflow: hidden`.

## When to use

- Choose one (or, with `multiple`, several) values from a fixed list of 5–50 options.
- Field needs the chosen value visible at all times.
- Form layouts where the trigger should match Input height for a tidy grid.

## When not to use

- Free-form text — use [Input](?path=/docs/form-input--docs) or [Autocomplete](?path=/docs/form-autocomplete--docs).
- Two-way list with filter — use [Listbox](?path=/docs/form-listbox--docs).
- Boolean toggle — use [Checkbox](?path=/docs/form-checkbox--docs) or [Switch](?path=/docs/form-switch--docs).

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair the trigger with `<Label for>` matching the `id` you set on `SelectTrigger`.
- **3.3.1 Error identification**: `error` sets `aria-invalid="true"`; reference helper text via `describedBy`.
- **2.4.7 Focus visible**: trigger draws a 2 px ring; items show a clear `data-highlighted` background.
- **2.5.8 Target size**: every trigger size meets the 24 × 24 px minimum; items are at least 24 px tall.
- **4.1.2 Name, role, value**: built on Reka UI primitives that emit the correct ARIA combobox / listbox semantics.

## Responsive behaviour

<Canvas of={SelectStories.InteractiveResponsive} />

The trigger is `w-full` and shrinks to its parent. Wrap in a sized container (`w-72`, `max-w-sm`) so very long values don't stretch wide layouts.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Pair every trigger with a <code>&lt;Label for&gt;</code>.
			</td>
			<td>Rely on placeholder as the only label.</td>
		</tr>
		<tr>
			<td>
				Use the <code>error</code> prop and a referenced helper text id for validation.
			</td>
			<td>
				Apply <code>class="border-destructive"</code> directly — it skips <code>aria-invalid</code>.
			</td>
		</tr>
		<tr>
			<td>Pick a size that matches surrounding Input fields.</td>
			<td>
				Override height with <code>class="h-12"</code> — propose a new size variant instead.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>multiple</code> when the user can choose more than one value.
			</td>
			<td>Stack two single Selects when the answer is a set.</td>
		</tr>
	</tbody>
</table>
