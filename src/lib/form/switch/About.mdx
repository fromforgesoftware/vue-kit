# Switch

Binary on/off control for settings that take immediate effect. Built on Reka UI's `SwitchRoot` primitive.

## When to use

- Toggle a single boolean setting that applies immediately (notifications, dark mode).
- Surface a feature flag inside a settings panel.
- Inline within a row where the change should be obvious without a save button.

## When not to use

- Multi-select inside a list — use [Checkbox](?path=/docs/form-checkbox--docs).
- Mutually exclusive choice — use [Radio](?path=/docs/form-radio--docs).
- Settings that require an explicit save — use Checkbox so the unsaved state reads as a form field.

## Anatomy

<Canvas of={SwitchStories.Default} />

A pill-shaped track with a circular thumb that slides between off (left) and on (right) positions. An invisible padding ring expands the click target to ≥ 24 × 24 px.

## Variants

<Canvas of={SwitchStories.Variants} />

Two states: unchecked (`false`) and checked (`true`). State is exposed via `data-state="checked|unchecked"` and `aria-checked`.

## Composition

<Canvas of={SwitchStories.WithLabel} />

Pair Switch with Label using `for` matching the switch's `id`. This:

- Makes clicking anywhere on the label row toggle the switch.
- Gives the switch a programmatic accessible name.
- Lets `getByLabelText` find the switch in tests.

<Canvas of={SwitchStories.Group} />

For a group of toggles, render one Switch per option inside its own Label.

## Accessibility (WCAG 2.2 AA)

- **2.5.8 Target size**: visible track is < 24 px tall but the `::before` ring expands the click target to ≥ 24 × 24 px. Composing with a Label inside a `flex` row gives a row-wide hit area that always meets the minimum.
- **4.1.2 Name, role, value**: Reka exposes `role="switch"` and `aria-checked`. Always pair with a Label or pass `ariaLabel` — never rely on adjacent text alone.
- **3.3.1 Error identification**: pass `error` to set `aria-invalid="true"` and use `describedBy` for an error message id.
- **2.1.1 Keyboard**: Space and Enter toggle the value. Tab moves focus.
- **2.4.7 Focus visible**: focus draws a 2 px ring + 2 px offset against the surrounding surface.
- **1.4.11 Non-text contrast**: the checked state uses `bg-primary` against `bg-input` for unchecked — both meet 3 : 1 against the background.

## Responsive behaviour

<Canvas of={SwitchStories.InteractiveResponsive} />

Switches don't change layout across breakpoints. The hit area is constant at every viewport.
