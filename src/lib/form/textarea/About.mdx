# Textarea

Multi-line text field. Mirrors Input tokens (border, ring, error tones) and adds optional auto-resize that grows the field with content.

## When to use

- Free-form long-form text: bio, comments, descriptions, change-notes.
- Form fields where the user needs more than one line.

## When not to use

- Single-line entry — use [Input](?path=/docs/form-input--docs).
- Rich text or markdown — use a dedicated rich-text editor.

## Accessibility (WCAG 2.2 AA)

- **2.4.6 Labels**: pair every textarea with a `<Label for>` that matches its id.
- **3.3.1 Error identification**: pass `error` to set `aria-invalid="true"` and use `describedBy` for the error message id.
- **3.3.2 Labels or instructions**: use a referenced helper element rather than placeholder for instructions.
- **2.4.7 Focus visible**: focus draws a 2 px ring at ≥ 3 : 1 contrast.
- **2.5.8 Target size**: every size meets the 24 × 24 px minimum.
- **iOS no-zoom**: `text-sm` (12 px Inter) prevents iOS Safari from zooming on focus.

## Responsive behaviour

<Canvas of={TextareaStories.InteractiveResponsive} />

The textarea is `w-full` and shrinks to its parent. Place it inside a max-width container so very long entries don't dominate wide layouts.
