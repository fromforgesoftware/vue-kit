# ToggleGroup

A connected row (or column) of [Toggle](?path=/docs/form-toggle--docs) items with single- or multi-select semantics. Items inherit the Toggle CVA so density and variants stay consistent. Built on Reka `ToggleGroup` — roving focus, arrow-key navigation, and `aria-pressed` are handled for you.

## When to use

- Mutually exclusive choice from a small visual set (text alignment, view mode).
- Multi-select formatting toolbar where each toggle stands on its own (Bold / Italic / Underline).

## When not to use

- More than ~5 options — use a [Select](?path=/docs/form-select--docs).
- Long text labels — use [RadioGroup](?path=/docs/form-radio--docs) for radios, [Tabs](?path=/docs/general-tabs--docs) for tabbed views.
- A single toggle — use [Toggle](?path=/docs/form-toggle--docs).

## Selection modes

`type="single"` enforces at most one active item. `type="multiple"` allows any combination — the value is `string[]`.

## Sizes

<Canvas of={ToggleGroupStories.Sizes} />

Apply the same `size` to every item in a group. Heights match the Input ramp (`sm` 28 px, `default` 32 px, `lg` 40 px).

## Orientation

<Canvas of={ToggleGroupStories.Vertical} />

Vertical layout swaps the divider from a left-border to a top-border and re-rounds the first/last items along the long axis. Arrow keys navigate along the chosen orientation.

## States

<Canvas of={ToggleGroupStories.Disabled} />

<Canvas of={ToggleGroupStories.ItemDisabled} />

Disable the whole group via `disabled` on `ToggleGroup`. Disable a single item via `disabled` on `ToggleGroupItem` — the rest stay interactive.

## Accessibility (WCAG 2.2 AA)

- **4.1.2 Name, role, value**: each item is a native `<button>` with `aria-pressed`. Always pass an `aria-label` (icon-only) or text content.
- **2.4.6 Headings and labels**: pass a descriptive `aria-label` on `ToggleGroup` (e.g. "View mode", "Text alignment").
- **2.5.8 Target size**: every size meets the 24×24 minimum on each item.
- **2.4.7 Focus visible**: focus draws a 2 px ring on the focused item.
- **2.1.1 Keyboard**: complete arrow / Home / End / Space / Enter keyboard model managed by the underlying roving-focus group.

## Responsive behaviour

<Canvas of={ToggleGroupStories.InteractiveResponsive} />

Groups are inline-flex; wrap them in `flex flex-wrap` if you stack multiple groups on the same row.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Pass a descriptive <code>aria-label</code> to <code>ToggleGroup</code>.
			</td>
			<td>Rely on a visual heading alone — the group needs its own programmatic name.</td>
		</tr>
		<tr>
			<td>
				Use the same <code>size</code> on every item.
			</td>
			<td>Mix sizes — the row will look uneven.</td>
		</tr>
		<tr>
			<td>Use ToggleGroup for ≤ 5 options.</td>
			<td>Use ToggleGroup for many options — switch to a Select.</td>
		</tr>
		<tr>
			<td>
				Use <code>type="single"</code> when exactly one option must be active.
			</td>
			<td>
				Use <code>type="multiple"</code> just to allow zero — single-select can also be cleared by
				clicking the active item.
			</td>
		</tr>
	</tbody>
</table>
