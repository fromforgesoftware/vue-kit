# Spacing & layout

Spacing follows Tailwind's default scale (`0.25rem` = 4 px increments). The library does not introduce custom spacing tokens — the standard `p-*`, `m-*`, `gap-*`, and `space-*` utilities are used everywhere.

**Mobile-first rule**: components define their mobile baseline first (no prefix), then upscale with `sm:`/`md:`/`lg:`. Use `max-sm:` only for genuinely mobile-specific patterns where the layout is fundamentally different (Dialog → bottom-sheet).

For complex composite controls (segment bar drag handles, slider thumbs), the visible affordance can be smaller than 24 × 24 as long as the **hit zone** is ≥ 24 × 24. Use a transparent expanded box (e.g. `::before` absolute positioning) — never sacrifice a11y for a tighter visual.

## Layout primitives

The library does not ship a Stack, Inline, or Box component — Tailwind's `flex`, `grid`, `gap`, and `space-*` utilities are sufficient and discoverable.

```html
<!-- Vertical stack -->
<div class="flex flex-col gap-3">…</div>

<!-- Horizontal inline -->
<div class="flex items-center gap-2">…</div>

<!-- Grid -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4">…</div>
```

## Don't

- Use arbitrary spacing (`p-[18px]`). Snap to the scale — propose a new step if the scale is missing it.
- Mix margin and padding for the same effect. Prefer `gap-*` on the parent.
- Override library component padding via `class=""` overrides in consumers. Add a `density` or `size` variant to the component instead.
