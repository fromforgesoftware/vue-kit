# Alert

Static, in-page message that surfaces information, warnings, errors, or successes that should remain visible until the user reads or dismisses them. For transient feedback after an action, use `Toast` instead.

## When to use

- Persistent context for a form ("This entity is synced — some fields are read-only").
- Page-level errors that block content ("We couldn't load this section").
- Inline guidance that requires the user's attention while they work.

## When not to use

- Transient action feedback — use [Toast](?path=/docs/general-toast--docs).
- Confirmations that block flow — use [AlertDialog](?path=/docs/general-alertdialog--docs).
- Field-level validation — use the field's `error` slot.

## Anatomy

<Canvas of={AlertStories.Default} />

An alert wraps a leading icon, a title, and a description. The container has `role="alert"` so assistive tech announces the message when it appears.

## With icon

Pass any Lucide icon via the `icon` prop. The icon inherits the variant colour — no per-variant overrides at the consumer.

## Title only

<Canvas of={AlertStories.TitleOnly} />

For short, one-line messages, omit the description.

## Dismissible

<Canvas of={AlertStories.Dismissible} />

Set `dismissible` to render a close button. The button:

- Has a 24 × 24 px touch target (WCAG 2.2 SC 2.5.8).
- Activates with `Enter` and `Space`.
- Exposes its purpose via `aria-label` (override with `dismissLabel`).
- Emits `dismiss`; consumers update their state.

```vue
<Alert dismissible dismiss-label="Dismiss notice" @dismiss="dismissed = true">
  <AlertTitle>Heads up</AlertTitle>
  <AlertDescription>...</AlertDescription>
</Alert>
```

## Composition

`Alert` is the container. `AlertTitle` and `AlertDescription` are slotted children — both pass through the alert's `size` so all three components stay aligned.

```vue
<Alert variant="warning" :icon="AlertTriangle">
  <AlertTitle>Heads up</AlertTitle>
  <AlertDescription>Something to look out for.</AlertDescription>
</Alert>
```

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: rendered as `<div role="alert">`. Title is an `<h5>`; description is a sibling `<div>`.
- **1.4.3 Contrast**: every variant pairs foreground and background to ≥ 4.5 : 1. Soft tones use 10 % tint with full-strength foreground colour.
- **2.1.1 Keyboard**: the dismiss button activates with `Enter` and `Space`.
- **2.4.7 Focus Visible**: focus ring on the dismiss button uses `outline-2 outline-offset-1`.
- **2.5.8 Target Size**: the dismiss button is 24 × 24 px minimum.
- **4.1.2 Name, Role, Value**: dismiss button always has an accessible name (`dismissLabel`).
- **4.1.3 Status Messages**: `role="alert"` triggers an immediate announcement when the element appears.

## Responsive behaviour

<Canvas of={AlertStories.InteractiveResponsive} />

Alerts are full-width by default. They wrap their content rather than truncating, so messages stay legible at every breakpoint.
