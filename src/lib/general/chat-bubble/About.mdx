# ChatBubble

Speech bubble used in chat / messaging surfaces. Three directions cover the most common patterns.

## When to use

- Chat / messaging surfaces (DM, support chat, agent inbox).
- Inline conversation summaries inside dashboards.
- AI assistant panels.

## When not to use

- For static comments — use a Card or list pattern.
- For notifications — use [Toast](?path=/docs/general-toast--docs) or [Alert](?path=/docs/general-alert--docs).
- For user-by-user messaging without role context — use a regular list.

## Without tail

<Canvas of={ChatBubbleStories.NoTail} />

For consecutive messages from the same author or for system messages, set `tail` to `false`.

## Loading

<Canvas of={ChatBubbleStories.Loading} />

When awaiting an upstream message (typing indicator, AI response), set `loading`. The bubble renders a spinner inside a polite live region.

## System message

<Canvas of={ChatBubbleStories.SystemMessage} />

Use `direction="system"` for centred, low-emphasis messages (joined / left, status changes). The tail is always hidden in this direction.

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: each bubble is a `<div role="group">` with an optional `aria-label="Message from {author}"`. Loading bubbles use `role="status"` for live announcement.
- **1.4.3 Contrast**: every direction × tone meets ≥ 4.5 : 1 against the bubble background.
- **2.4.6 Headings and Labels**: pass `authorLabel` so screen readers announce the message author.
- **4.1.3 Status Messages**: loading bubbles use `role="status"`.

## Responsive behaviour

<Canvas of={ChatBubbleStories.InteractiveResponsive} />

Bubbles are capped at 75 % of their container width and wrap text rather than overflow. Long URLs use `break-words`.
