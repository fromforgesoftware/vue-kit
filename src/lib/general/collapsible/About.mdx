# Collapsible

A single open/closed disclosure section. Built on Reka UI primitives — keyboard, ARIA, and animation handled internally. Use it to hide secondary content (advanced settings, an inline help block, a long detail) behind a single toggle. For multiple sections, reach for [Accordion](?path=/docs/general-accordion--docs) instead.

## When to use

- "Show advanced options" toggles inside a form.
- A single inline help block that defaults closed.
- A details/summary block where only one trigger exists.

## When not to use

- Several stacked disclosures — use [Accordion](?path=/docs/general-accordion--docs).
- Modal-style disclosure that blocks the surface — use a [Dialog](?path=/docs/general-dialog--docs) or [Drawer](?path=/docs/general-drawer--docs).
- Hover-revealed help — use a [Tooltip](?path=/docs/general-tooltip--docs).

## Anatomy

<Canvas of={CollapsibleStories.Default} />

The trigger is a real `<button>` exposing `aria-expanded`. Content is animated open/close via the `collapsible-down` and `collapsible-up` keyframes.

## Composition

```vue
<Collapsible variant="default" size="default">
  <CollapsibleTrigger>Show advanced options</CollapsibleTrigger>
  <CollapsibleContent>Body content.</CollapsibleContent>
</Collapsible>
```

## Variants

<Canvas of={CollapsibleStories.Variants} />

- `ghost` — borderless. Use inside sidebars, filter panels, or anywhere chrome already provides separation.

## Accessibility (WCAG 2.2 AA)

- **1.4.3 Contrast**: trigger and content text use `text-foreground` / `text-muted-foreground` — both clear 4.5 : 1 against `bg-card`.
- **1.4.11 Non-text contrast**: borders and the focus ring clear 3 : 1 against the surrounding surface.
- **2.1.1 Keyboard**: the trigger is a real `<button>` — Space and Enter toggle, focus follows natural tab order.
- **2.4.7 Focus visible**: a 3 px inset ring at `--ring` is applied on `:focus-visible`.
- **2.5.8 Target size**: trigger meets the 24×24 px minimum at every size and viewport (default size is 44 px tall; `sm` is 36 px).
- **4.1.2 Name, Role, Value**: trigger exposes `role="button"`, `aria-expanded`, and `aria-controls` pointing at the content region.

## Responsive behaviour

<Canvas of={CollapsibleStories.InteractiveResponsive} />

The collapsible is `w-full` and never causes horizontal overflow. Trigger labels wrap; long unbreakable strings (URLs, IDs) need `break-all` or `truncate` at the consumer.
