# Context Menu

A right-click (or long-press on touch) menu anchored to a region. Same composition as Dropdown Menu, but the trigger is the area itself instead of a button. Built on Reka UI — content portals to `document.body`. Supports submenus, checkbox items, radio groups, and a destructive variant.

## When to use

- Schedule cells, calendar events, table rows where a right-click is expected.
- Power-user shortcuts complementing existing visible buttons.
- Editor surfaces (canvas, document) where the context menu mirrors keyboard actions.

## When not to use

- The menu is the only way to perform an action — non-power users won't discover it. Pair with a visible [Dropdown Menu](?path=/docs/general-dropdown-menu--docs).
- Toolbar-style top-of-app navigation — use [Menubar](?path=/docs/general-menubar--docs).
- Hover-only previews — use [Hover Card](?path=/docs/general-hover-card--docs).

## Accessibility (WCAG 2.2 AA)

- **2.4.3 Focus order**: focus moves into the menu on open and returns to the page after close.
- **2.4.7 Focus visible**: items keep the standard `:focus-visible` ring.
- **2.1.1 Keyboard**: a context menu is **discoverable only by mouse** — always provide an alternative path (a visible button, a Dropdown Menu) so keyboard-only users can reach the same actions.
- **4.1.2 Name, Role, Value**: items use `role="menuitem"` automatically.
- **2.5.8 Target size**: items render at minimum 24 px tall.

> A context menu is a power-user shortcut, not a discoverable affordance. Never make it the _only_ path to an action.

## Responsive behaviour

<Canvas of={ContextMenuStories.InteractiveResponsive} />

The menu is portaled to `document.body` and never affects parent layout. On touch, long-press triggers the menu — keep targets large enough to long-press accurately.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>Mirror visible actions in the context menu so power users can right-click.</td>
			<td>
				Make the context menu the only way to delete or rename — keyboard users will be stuck.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>variant="destructive"</code> for delete and remove actions.
			</td>
			<td>
				Override colours with <code>class="text-destructive"</code>.
			</td>
		</tr>
		<tr>
			<td>
				Group related items with <code>ContextMenuLabel</code> + <code>ContextMenuSeparator</code>.
			</td>
			<td>Stuff a 20-item list into one context menu.</td>
		</tr>
		<tr>
			<td>Test long-press behaviour on touch — many users discover context menus there.</td>
			<td>
				Use a <code>div</code> with no role as the trigger — wrap an interactive region.
			</td>
		</tr>
	</tbody>
</table>
