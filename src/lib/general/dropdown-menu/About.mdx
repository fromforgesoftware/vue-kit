# Dropdown Menu

A list of actions or options that opens when the user clicks a button. Supports submenus, checkbox items, radio groups, keyboard shortcuts, and a destructive variant for delete actions. Built on Reka UI — content portals to `document.body`.

## When to use

- Action menus on a row, card, or toolbar (Edit / Duplicate / Delete).
- Account / user menus with sign out, profile, switch tenant.
- Toggling visible columns, sorting, filters.

## When not to use

- Right-click context actions — use [Context Menu](?path=/docs/general-context-menu--docs).
- Application-wide horizontal menus (File / Edit / View) — use [Menubar](?path=/docs/general-menubar--docs).
- A select that captures one value into a form — use Select.
- Plain help text or hover previews — use Tooltip or Hover Card.

## Variants

<Canvas of={DropdownMenuStories.Variants} />

`DropdownMenuItem` accepts `variant="destructive"` for delete or remove actions. The variant tints the text and any icon red and shifts the focus background to a soft destructive tint — never override the colour with `class`.

## Accessibility (WCAG 2.2 AA)

- **2.4.3 Focus order**: focus moves into the menu on open and returns to the trigger on close.
- **2.4.7 Focus visible**: items keep the standard `:focus-visible` ring; focused items also pick up an accent background.
- **2.1.1 Keyboard**: full menu is operable from the keyboard — open, navigate, activate, close.
- **4.1.2 Name, Role, Value**: trigger keeps its own role and accessible name; items use `role="menuitem"`, `role="menuitemcheckbox"`, `role="menuitemradio"` automatically.
- **2.5.8 Target size**: items render at minimum 24 px tall thanks to `min-h-6` on item variants.

> Pair every icon-only `DropdownMenuTrigger` with `aria-label` on its inner button — the trigger has no accessible name otherwise.

## Responsive behaviour

<Canvas of={DropdownMenuStories.InteractiveResponsive} />

The menu is portaled to `document.body` and never affects parent layout. Triggers should respect their container's width.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Use <code>variant="destructive"</code> for Delete and Remove actions.
			</td>
			<td>
				Override colours with <code>class="text-destructive"</code>.
			</td>
		</tr>
		<tr>
			<td>
				Group related items with <code>DropdownMenuLabel</code> + <code>DropdownMenuSeparator</code>
				.
			</td>
			<td>List 12+ unrelated actions in one menu — split into a sub-menu or separate Popover.</td>
		</tr>
		<tr>
			<td>
				Pair an icon-only trigger with <code>aria-label</code>.
			</td>
			<td>Rely on the menu items as the trigger's accessible name.</td>
		</tr>
		<tr>
			<td>
				Keep shortcuts visible with <code>DropdownMenuShortcut</code> for power users.
			</td>
			<td>Show shortcuts that aren't actually wired up.</td>
		</tr>
	</tbody>
</table>
