# EmptyState

Page-level placeholder shown when a list, table, or panel has no data yet. Always pair with a clear next-step action.

## When to use

- A new account that hasn't created anything yet ("No employees yet — invite your first").
- A filter that returns no results ("No matches — try widening the date range").
- A panel awaiting first sync.

## When not to use

- For loading states — use [Skeleton](?path=/docs/general-skeleton--docs).
- For error states — use [Alert](?path=/docs/general-alert--docs) or `ForbiddenState`.
- For absence inside dense data tables — render a single "No matches" row instead.

## Anatomy

<Canvas of={EmptyStateStories.Default} />

A vertical stack: icon, title (`<h3>`), description, and one or two action slots. The container has `role="region"` (via `<section>`) with the title as `aria-label`.

## With actions

<Canvas of={EmptyStateStories.WithActions} />

Use the default slot for one or two buttons. Pair primary + outline for "create + refresh" or "create + import".

## With action cards

<Canvas of={EmptyStateStories.WithActionCards} />

For multi-step onboarding, use the `actions` slot with `EmptyStateAction` cards. Each card carries its own title, description, and trigger.

## Tones

<Canvas of={EmptyStateStories.Tones} />

`default` is transparent — best for use inside cards. `muted` adds a soft tinted background — best for use directly on the page surface.

## No icon

<Canvas of={EmptyStateStories.NoIcon} />

For minimal contexts (inline forms, narrow sidebars), omit the icon.

## Custom icon slot

<Canvas of={EmptyStateStories.CustomIconSlot} />

Override the icon area entirely with the `icon` slot — useful for stacked glyphs, illustrations, or branded imagery.

## Illustration

<Canvas of={EmptyStateStories.Illustration} />

For full-page empty states (especially in DataTables and feature landing pages), pass an `illustration` name instead of an `icon`. Light/dark variants are loaded from `@fromforgesoftware/ts-kit/assets/chart-illustrations/` and swapped automatically via the `[data-theme]` attribute. Available names are exported as `ILLUSTRATION_NAMES`.

## Composition

```vue
<EmptyState
	illustration="empty-datatable"
	title="No employees yet"
	description="Get started by adding your first employee to the system."
>
  <Button>
    <Icon :icon="Plus" size="sm" />
    Add employee
  </Button>
</EmptyState>
```

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: rendered as `<section>` with `aria-label` set to the title; the title itself is an `<h3>`.
- **1.4.3 Contrast**: title uses `text-foreground`; description uses `text-muted-foreground` (≥ 4.5 : 1 against the surface).
- **2.4.7 Focus Visible**: action buttons inherit the system focus ring.
- **2.5.8 Target Size**: action buttons must be at least the default `Button` size (≥ 36 px) — never use `size="xs"` here.

## Responsive behaviour

<Canvas of={EmptyStateStories.InteractiveResponsive} />

- Action cards shift from a 2-column grid to a stack at `< sm` (640 px).
- Inline actions wrap in a `flex-wrap` row.
