# ForbiddenState

Full-page placeholder shown when access is denied or content is missing. Built on top of `EmptyState` with permission-domain presets.

## When to use

- A user lands on a page their role can't access (`403 Forbidden`).
- A resource has been deleted or never existed (`404 Not Found`).
- An unrecoverable error blocks the current view.
- A feature is gated behind an admin-only toggle.

## When not to use

- For zero-data placeholders — use [EmptyState](?path=/docs/general-emptystate--docs).
- For inline form errors — use the field's error slot.
- For transient errors — use [Alert](?path=/docs/general-alert--docs) or [Toast](?path=/docs/general-toast--docs).

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: rendered as `<section aria-label="…">` with the title as `<h3>`.
- **1.4.3 Contrast**: title `text-foreground`, description `text-muted-foreground`, both ≥ 4.5 : 1.
- **2.4.7 Focus Visible**: action buttons inherit the system focus ring.
- **3.3.1 Error Identification**: title and description describe the error in plain language; consumers should pair with logging at the route layer.
