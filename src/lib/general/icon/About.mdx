# Icon

Thin wrapper around the [Lucide](https://lucide.dev) icon set that applies the design-system size scale and stroke width. Always pair with `aria-hidden="true"` when decorative — keep meaningful icons accessible by setting `role` and `aria-label` on the parent.

## When to use

- Inline glyphs in buttons, alerts, badges, list items.
- Iconography in toolbars and menus.
- Empty-state and forbidden-state illustrations.

## When not to use

- For brand glyphs that aren't in Lucide — use a dedicated SVG file.
- For decorative imagery beyond inline icons — use a real image asset.

## Anatomy

<Canvas of={IconStories.Default} />

A `<component :is="icon">` rendered with the size class, configurable stroke width, and your overrides via `class`.

## Stroke width

<Canvas of={IconStories.StrokeWidth} />

The default stroke width is `2`. Drop to `1.5` for delicate icons in dense layouts; raise to `2.5` or `3` for emphasis.

## Colour

<Canvas of={IconStories.Colored} />

Icons inherit `currentColor`. Colour them via Tailwind text utilities on the wrapping element (`text-primary`, `text-muted-foreground`, …).

## Accessibility (WCAG 2.2 AA)

- **1.1.1 Non-text Content**: icons are decorative by default. When the icon conveys meaning (e.g. a close button), give the wrapping element an `aria-label`.
- **1.4.3 Contrast**: when the icon carries meaning, ensure the colour meets ≥ 3 : 1 against the background.
- **2.5.8 Target Size**: an icon by itself is not a target. When wrapped in a button, the button must meet 24 × 24 px.

## Responsive behaviour

<Canvas of={IconStories.InteractiveResponsive} />

Icons use absolute pixel sizes that don't change across breakpoints. If your layout needs adaptive sizing, switch the `size` prop manually based on a breakpoint signal.
