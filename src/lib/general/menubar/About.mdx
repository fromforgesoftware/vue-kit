# Menubar

A horizontal application menu — File / Edit / View. Use sparingly: most modern apps prefer one Dropdown Menu plus an in-app command palette. Reach for Menubar in genuinely menu-driven editors (canvas, document, schedule editor) where users expect a top-of-app bar.

## When to use

- Editor surfaces with many keyboard shortcuts and a long-tail of actions.
- Apps mirroring desktop conventions (drawing, video, IDE).

## When not to use

- General CRUD apps — use a single [Dropdown Menu](?path=/docs/general-dropdown-menu--docs) on the top bar.
- Mobile-first surfaces — a Menubar at the top is rarely usable on small screens.
- Right-click actions — use [Context Menu](?path=/docs/general-context-menu--docs).

## Accessibility (WCAG 2.2 AA)

- **2.4.3 Focus order**: focus follows visual order along the bar; opening a menu moves focus into it; closing returns focus to the trigger.
- **2.4.7 Focus visible**: every trigger and item keeps the standard `:focus-visible` ring.
- **2.1.1 Keyboard**: full bar is operable from the keyboard with arrow navigation, type-ahead, Enter/Escape.
- **4.1.2 Name, Role, Value**: triggers use `role="menuitem"`, content uses `role="menu"`, items inside use `role="menuitem"` / `menuitemcheckbox` / `menuitemradio` automatically.
- **2.5.8 Target size**: items render at minimum 24 px tall.

> Always pair Menubar with discoverable shortcuts in a Command palette — keyboard-only users shouldn't need to drill through three menus to reach an action.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>Use Menubar for editor-style apps where users expect File / Edit / View at the top.</td>
			<td>Use Menubar for general dashboards or settings pages.</td>
		</tr>
		<tr>
			<td>Pair every menu item with a keyboard shortcut for power users.</td>
			<td>Stuff 30+ items into a single menu — split with submenus.</td>
		</tr>
		<tr>
			<td>
				Use <code>variant="destructive"</code> for Exit / Delete.
			</td>
			<td>
				Override colours via <code>class="text-destructive"</code>.
			</td>
		</tr>
		<tr>
			<td>Hide the bar on mobile and surface a single Dropdown Menu instead.</td>
			<td>Force a long Menubar to wrap onto two lines on a phone.</td>
		</tr>
	</tbody>
</table>
