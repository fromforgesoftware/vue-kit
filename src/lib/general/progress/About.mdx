# Progress

Determinate progress bar that visualises a known percentage. For unknown durations (waiting on a request) use `indeterminate` mode or a `Spinner`.

## When to use

- Multi-step flows where the total can be measured (onboarding, file upload).
- Capacity / quota usage (storage, seats, API calls).
- Long-running batch jobs where the server reports percentage.

## When not to use

- For very short waits (< 200 ms) — flicker beats progress.
- For unknown-duration tasks — use indeterminate mode or `Spinner`.
- For a series of parallel tasks — use multiple bars or a list with status.

## Anatomy

<Canvas of={ProgressStories.Default} />

A track and an indicator. The indicator's transform reflects the percentage; the track tints the variant colour at low alpha so colour is consistent at every value.

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: rendered by Reka UI as `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax` (or `aria-valuenow` omitted when indeterminate).
- **1.4.3 Contrast**: indicator colour pairs with a low-alpha track of the same hue — visible at every value.
- **2.2.2 Pause, Stop, Hide**: indeterminate animation uses CSS, respecting `prefers-reduced-motion`.
- **4.1.3 Status Messages**: pair with a labelled `<div role="status">` if the percentage should be announced as it changes.

## Responsive behaviour

<Canvas of={ProgressStories.InteractiveResponsive} />

Progress is full-width by default. Wrap in a constrained container to limit its width.
