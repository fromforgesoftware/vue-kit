# Spinner

Indeterminate progress indicator. Communicates that the system is working but cannot estimate completion.

## When to use

- Short waits inside a button or form (saving, verifying, fetching dropdown options).
- Loading states where the layout is unknown — if you know the shape, prefer `Skeleton`.
- Inline loading next to a label (`Spinner` + "Loading employees…").

## When not to use

- For long-running operations where progress can be measured — use `Progress`.
- For full-page initial loads where structure is known — use `Skeleton`.
- As a permanent part of the UI — spinners imply a transient state.

## Anatomy

<Canvas of={SpinnerStories.Default} />

A `<svg>` with `role="status"` and an accessible label, animated via `animate-spin`.

## With text

<Canvas of={SpinnerStories.WithText} />

Pair with a label for inline loading. The label can either repeat the spinner's `label` prop (visual) or carry richer context.

## In a button

<Canvas of={SpinnerStories.InButton} />

Inside a button, use `tone="current"` so the spinner inherits the button text colour automatically — no per-variant overrides.

```vue
<Button :disabled="saving">
  <Spinner v-if="saving" size="sm" tone="current" />
  Save
</Button>
```

## Accessibility (WCAG 2.2 AA)

- **1.1.1 Non-text Content**: the SVG carries `role="status"` and an `aria-label` (override via `label`). A `<title>` element backs the label for tools that read SVG titles.
- **2.2.2 Pause, Stop, Hide**: the rotation is CSS-driven (`animate-spin`) and respects `prefers-reduced-motion` automatically.
- **4.1.3 Status Messages**: `role="status"` triggers a polite live-region announcement when the spinner appears.

## Responsive behaviour

<Canvas of={SpinnerStories.InteractiveResponsive} />

Spinners use absolute pixel sizes — they do not change across breakpoints. If your layout needs adaptive sizing, switch the `size` prop manually based on a breakpoint signal.
