# Splitter

Drag-to-resize panel divider for two-or-three-pane layouts (sidebars, master/detail, IDE-style consoles). Built on Reka UI's `SplitterGroup` — keyboard navigation and ARIA semantics are wired internally.

## When to use

- Master/detail layouts where the user benefits from controlling the relative sizes (lists vs. preview).
- Sidebar + main content where the sidebar may be wider on desktops.
- IDE-style three-panel layouts (sidebar, main, console) — compose by nesting.

## When not to use

- Fixed layouts where the user does not need to resize — use a plain flex container.
- Modal-style overlays — use [Dialog](?path=/docs/general-dialog--docs) or [Drawer](?path=/docs/general-drawer--docs).
- Stacked card lists — use a regular column layout with consistent gaps.

## Anatomy

<Canvas of={SplitterStories.Default} />

A splitter is composed of `Splitter` (root), one or more `SplitterPanel` regions, and `SplitterResizeHandle` separators between adjacent panels.

## Composition

```vue
<Splitter direction="horizontal">
  <SplitterPanel :default-size="30" :min-size="20" surface="card">Sidebar</SplitterPanel>
  <SplitterResizeHandle />
  <SplitterPanel :default-size="70" :min-size="40" surface="card">Main</SplitterPanel>
</Splitter>
```

## Nested splitters

<Canvas of={SplitterStories.NestedSplitters} />

Compose horizontally and vertically by nesting. The most common shape — sidebar + main + console — is built by placing a vertical splitter inside the right panel of a horizontal splitter.

## Collapsible panels

<Canvas of={SplitterStories.Collapsible} />

Set `collapsible` on a `SplitterPanel` to allow it to snap closed when dragged below `min-size`. The component emits `collapse` and `expand` so the consumer can update icons or persist state.

## Accessibility (WCAG 2.2 AA)

- **2.5.7 Dragging Movements**: every drag interaction has a keyboard alternative — the handle accepts arrow keys to nudge the boundary in single-percent steps.
- **2.5.8 Target Size**: the handle's hit area is ≥ 24×24 px via a transparent `::before` expansion zone, regardless of which `thickness` is set.
- **4.1.2 Name, Role, Value**: the handle has `role="separator"` (Reka), an explicit `aria-label` (default `"Resize panels"` — override via the `aria-label` prop), and `aria-valuenow` reflecting the current percentage of the leading panel.
- **2.4.7 Focus Visible**: a 2 px focus ring is applied to the handle on `:focus-visible`. The visible rule also tints to the ring colour while focused or being dragged.
- **1.4.11 Non-text Contrast**: the rule uses `bg-border`, which clears 3:1 against `bg-background` and `bg-card`.

## Responsive behaviour

<Canvas of={SplitterStories.InteractiveResponsive} />

The Splitter itself fills its parent container and never exceeds it. Consumers should switch to a stacked layout on narrow viewports (`v-if="isDesktop"`) — Splitter is a desktop-class control and dragging on touch viewports requires a larger hit area than the WCAG minimum.
