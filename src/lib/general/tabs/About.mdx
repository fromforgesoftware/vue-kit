# Tabs

Switch between mutually exclusive content panels at the same level of hierarchy. Built on Reka UI primitives — keyboard navigation, ARIA roles, and the WAI-ARIA tab pattern are handled internally. The animated indicator slides between active triggers.

## When to use

- A handful of related views over the same data (Overview / Activity / Settings).
- Switching between report axes or filtered cuts of one dataset.
- Settings groups within a page or drawer.

## When not to use

- For navigation between routes — use [Breadcrumb](?path=/docs/general-breadcrumb--docs) and `<RouterLink>`. Tabs change panels, not URLs.
- For step-by-step wizards — use [Stepper](?path=/docs/general-stepper--docs).
- When all panels need to be visible at once — use sections with headings.

## Composition

```vue

  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>

  </TabsList>
  <TabsContent value="overview">…</TabsContent>
  <TabsContent value="activity">…</TabsContent>
</Tabs>
```

## Orientations

<Canvas of={TabsStories.Orientations} />

`vertical` lays the list down the side. The indicator orientation flips automatically. Use in settings layouts or wizard sidebars.

## Manual activation

<Canvas of={TabsStories.ManualActivation} />

By default (`activationMode="automatic"`), arrow keys both move focus and activate the new tab. Set `activationMode="manual"` when each panel triggers a network request or expensive render — focus moves on arrows, but the user must press <kbd>Enter</kbd> or <kbd>Space</kbd> to activate.

## Accessibility (WCAG 2.2 AA)

- **1.4.3 Contrast**: active and inactive trigger text both clear 4.5 : 1 on their respective backgrounds.
- **1.4.11 Non-text contrast**: the indicator and focus ring clear 3 : 1 against the surrounding surface.
- **2.1.1 Keyboard**: every trigger is a real `<button>` with the WAI-ARIA tab pattern (`role="tab"`, `aria-selected`, `aria-controls`).
- **2.4.3 Focus order**: roving focus — only one tab stop per tablist. Inactive tabs have `tabindex="-1"`.
- **2.4.7 Focus visible**: a 3 px ring at `--ring` is applied on `:focus-visible`.
- **2.5.8 Target size**: every trigger meets the 24×24 px minimum at every size and viewport.
- **4.1.2 Name, Role, Value**: triggers expose `role="tab"` with `aria-selected` and `aria-controls` linking to the panel; panels expose `role="tabpanel"` with `aria-labelledby` pointing at their trigger.

## Responsive behaviour

<Canvas of={TabsStories.InteractiveResponsive} />

The `TabsList` is `overflow-x-auto` with hidden scrollbars (`scrollbar-none`) — when there are too many tabs to fit, the list scrolls horizontally rather than wrapping to a second row. The active tab is always reachable via arrow keys.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>Keep tab labels short (1–2 words).</td>
			<td>Use full sentences as labels — they break the layout on narrow viewports.</td>
		</tr>
		<tr>
			<td>
				Use <code>activationMode="manual"</code> for tabs that load remote data.
			</td>
			<td>Trigger expensive loads on every arrow-key press in automatic mode.</td>
		</tr>
		<tr>
			<td>Pick one variant per surface.</td>
			<td>
				Mix <code>underlined</code> and <code>pill</code> in the same view — pick one and stay
				consistent.
			</td>
		</tr>
		<tr>
			<td>Use Tabs for view-switching at the same hierarchy.</td>
			<td>Use Tabs for routes — change the URL with a navigation primitive instead.</td>
		</tr>
