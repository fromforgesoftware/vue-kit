# Virtual Scroll Area

A windowed scroll container backed by `@tanstack/vue-virtual`. Renders only the rows currently in (or near) the viewport, so 10 000+ items stay smooth on every device. Wraps Reka UI's `ScrollArea` so the styled scrollbar matches the rest of the system.

## When to use

- Rendering more than ~200 rows where each row is non-trivial (avatar, status chip, action button).
- An infinite-scroll list where the upstream API streams pages — pair with `@scroll-end`.
- A picker where the underlying data is large but the visible window is small (drop-down with all employees, all timezones, etc.).

## When not to use

- The list will always be ≤ 100 simple rows — a plain `<ul>` inside a `ScrollArea` is simpler and avoids measurement.
- The list is the page itself (no fixed height) — virtualization needs a constrained scroll container.
- Items have wildly different heights AND must support keyboard navigation between them — reach for a real list/grid component first.

## Anatomy

<Canvas of={VirtualScrollAreaStories.Default} />

The root is a styled `ScrollAreaRoot`. The viewport hosts an absolutely-positioned spacer the height of the entire data set; rendered rows are translated into place. The default slot receives `{ item, index }`.

## Accessibility (WCAG 2.2 AA)

- **1.4.11 Non-text contrast**: scrollbar thumb meets 3 : 1 against the track.
- **2.1.1 Keyboard**: the viewport is a native scroll container — `Page Up` / `Page Down` / `Home` / `End` and arrow keys all work when focused.
- **2.4.7 Focus visible**: focus ring is rendered by Reka's `ScrollAreaRoot`; rendered slot content is responsible for its own focus styles.
- **2.5.7 Dragging movements**: scrolling has a keyboard alternative (see above) — see `InteractiveKeyboardAlternative`.
- **2.5.8 Target size**: scrollbar thumb is taller than 24 px on every standard viewport.
- **4.1.3 Status messages**: when feeding `loading` upstream, announce it with `role="status"` outside the virtual list — the virtualized children are not always in the live region.

## Responsive behaviour

<Canvas of={VirtualScrollAreaStories.InteractiveResponsive} />

The container width follows whatever you set on `class`. The list never causes horizontal overflow — set the inner row's `class` to `truncate` or `line-clamp-N` when text is unbounded.

## Do / Don't

		<tr>

				Set <code>estimateSize</code> close to the actual median row height — saves the first
				measurement pass.
			</td>
			<td>
				Leave <code>estimateSize</code> at the default if your rows are 80 px tall — you'll
				over-render.
			</td>
		</tr>
		<tr>
			<td>
				Constrain the container with explicit <code>height</code> + <code>width</code> classes.
			</td>
			<td>Let the root grow to its content — virtualization needs a bounded viewport.</td>
		</tr>
		<tr>
			<td>
				Use <code>@scroll-end</code> for infinite-scroll loading.
			</td>
			<td>
				Listen for <code>scroll</code> on the inner viewport directly — bypasses the bounds check.
			</td>
		</tr>
		<tr>
			<td>Render keyboard-focusable controls inside rows for primary actions.</td>
			<td>Rely on hover-only affordances — virtualization tears them down on scroll.</td>
		</tr>
	</tbody>
