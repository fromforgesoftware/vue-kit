# Activity Timeline

A vertical, chronologically grouped feed of typed activity events: status changes, emails, attribute edits, calendar events, and merges. Designed for entity detail pages (deal, contact, ticket).

## When to use

- Show recent history for an entity grouped by time bucket ("Today", "This week").
- Surface mixed-type activity (system events + user actions) in one stream.
- Provide drill-down into bursts of similar events via expandable groups.

## When not to use

- High-frequency real-time logs — use a tabular log component.
- Notifications inbox — use a dedicated `Notification` list with read/unread state.

## Expandable groups

<Canvas of={ActivityTimelineStories.ShortFeed} />

When a burst of similar events would clutter the feed, group them under an `expandable_group` item. The toggle is a real `<button>` with `aria-expanded` reflecting state — it switches between "Show N more items" and "Hide N items" as it's pressed, and is reachable via Tab + activated by Enter / Space.

## Attribute change diff

<Canvas of={ActivityTimelineStories.AttributeChange} />

`attribute_change` rows show a one-line summary ("Editor has changed 2 attributes") with a chevron. Clicking the row reveals each changed attribute as `oldValue → newValue` with a strike-through on the old value.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>Group items into time buckets ("Today", "Yesterday", "This week")</td>
			<td>Render hundreds of ungrouped items in one list</td>
		</tr>
		<tr>
			<td>
				Use <code>expandable_group</code> for runs of 3+ similar events
			</td>
			<td>Show every email/comment inline by default</td>
		</tr>
		<tr>
			<td>
				Provide a clear <code>timestamp</code> on every item
			</td>
			<td>Rely on visual order alone for chronology</td>
		</tr>
	</tbody>
</table>
