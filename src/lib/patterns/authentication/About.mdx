# Authentication

A set of card-based screens that compose the full authentication flow: sign-in, register, forgot password, reset password, OTP verification, and email verification. Each screen is a self-contained Vue component built on top of the shared `AuthCard` shell, so consumers can drop them into Keycloak templates, in-app modal flows, or standalone routes.

## When to use

- Consumer-facing sign-up / sign-in routes where the entire screen is the auth surface.
- Keycloak login templates rendered inside an iframe or full-page redirect.
- In-app re-authentication prompts (OTP step before approving destructive actions).

## When not to use

- Sub-second confirmation flows — use a [Dialog](?path=/docs/general-dialog--docs) with a single password field.
- API-key entry / settings — use a generic form, not the auth shell.

## Composition

All screens share `AuthCard`, which gives them:

- Centred logo slot (defaults to a polygon brand mark; consumers override via `#logo`).
- Title + subtitle slots wired through `#title` / `#subtitle`.
- A consistent card padding, border, and rounded surface across screens.

		</tr>
