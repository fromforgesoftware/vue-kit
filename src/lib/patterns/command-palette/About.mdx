# Command Palette

Global search and command launcher inspired by Raycast and Linear. Opens via a `CommandPaletteTrigger` button (or `⌘K`) and surfaces a unified searchable list of routes, actions, and entities.

## When to use

- Global app search bound to `⌘K` / `Ctrl+K`.
- Keyboard-first action launcher for power users.
- Multi-mode search (commands with `>` prefix, people with `@` prefix).

## When not to use

- A simple type-ahead within a single field — use `Combobox`.
- Top-of-page persistent search bar — use `InputSearch`.

## Trigger button

<Canvas of={CommandPaletteStories.WithTrigger} />

`CommandPaletteTrigger` renders a search-shaped button with the `⌘K` shortcut. Wrap any palette content as the slot child — the trigger handles opening, focus trapping, and Escape-to-close via `CommandPaletteDialog`.

## Registry-driven filtering

<Canvas of={CommandPaletteStories.WithRegistry} />

Pass a custom `filter` function to search across labels and `keywords` aliases. Items expose their keywords on the DOM so consumers can read them in custom filter implementations.

## Prefix modes

<Canvas of={CommandPaletteStories.PrefixModes} />

Use `v-model:search` to react to prefix tokens (`/` for commands, `@` for people). The consumer toggles which groups render based on the leading character.

## Loading state

<Canvas of={CommandPaletteStories.LoadingState} />

`CommandPaletteList` accepts `loading` to render a placeholder while async results are fetched.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>Group items by category (Recent / Navigation / Actions)</td>
			<td>Render a flat list of 100+ items with no grouping</td>
		</tr>
		<tr>
			<td>
				Provide <code>keywords</code> aliases for synonyms (e.g. "people" → Employees)
			</td>
			<td>Rely solely on the visible label for matching</td>
		</tr>
		<tr>
			<td>
				Wire <code>@select</code> to a navigation or action handler and call <code>close</code>
			</td>
			<td>Mutate state inside the palette without closing — leaves the user stuck</td>
		</tr>
		<tr>
			<td>
				Use <code>CommandPaletteEmpty</code> to handle no-match state
			</td>
			<td>Render an empty list with no feedback</td>
		</tr>
	</tbody>
</table>
