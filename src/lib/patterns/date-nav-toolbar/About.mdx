# Date Nav Toolbar

A standardised toolbar for calendar / timeline-style views. Combines a view toggle, a tightly-grouped prev / date-picker / next cluster, and an `actions` slot. Used by both `EventCalendar` and the app's `TimelineView` so navigation feels identical across surfaces.

## When to use

- Any page that lets the user switch between Day / Week / Month / Year views over a `ForgeDate`.
- Surfaces that need a date-picker as the primary date control (clicking the label opens the picker, presets included).

## When not to use

- For a single fixed-period view (no view switcher) — drop in a plain `DatePicker` instead.
- For mini calendars embedded inside cards — the toolbar's affordances are too heavy.

## Picker types

Picker auto-derives from the `view` value: `month` → `MonthPicker`, `year` → `YearPicker`, anything else → `DatePicker`. Override with the `picker-type` prop when your view name doesn't match.

<Canvas of={DateNavToolbarStories.MonthPickerView} />
<Canvas of={DateNavToolbarStories.YearPickerView} />

## Right-side actions

Use the `#actions` slot for filters, "New event" buttons, group selectors — anything page-specific.

<Canvas of={DateNavToolbarStories.WithActions} />

## Two-way binding

Both `view` and `selectedDate` support `v-model`. `prev` / `next` are emitted as plain events so the parent decides the step size (a day, a week, a month — depends on the active view).

## Responsive behaviour

The outer wrapper is `flex flex-wrap gap-3`. On narrow viewports the actions row drops below the navigation cluster naturally; both clusters keep their internal grouping.
