# EventCalendar

A multi-view calendar surface (day, week, month, year) for displaying time-bound events. Renders an internal navigation header that mirrors the timeline view: a view-switcher on the left, a tightly-grouped previous / date-picker / next cluster, and an `actions` slot on the right. The date label is an interactive `DatePicker` (or `MonthPicker` / `YearPicker` depending on view) — clicking it opens a calendar with presets, including "Today".

## When to use

- Browsing scheduled events across day, week, month, or year.
- Building a consumer-controlled calendar where the parent owns `view` and `selectedDate` via `v-model`.
- Surfacing event hover popovers or click-to-edit affordances against a date grid.

## When not to use

- Editing shift schedules per employee — use [`ScheduleTimeline`](?path=/docs/patterns-scheduletimeline--docs).
- Pure date-picking — use a date input or calendar picker primitive.

## Responsive behaviour

The calendar root never overflows its container horizontally. The day/week grid scrolls vertically inside its own scroll area; the month/year layouts shrink columns to fit.

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Drive <code>view</code> and <code>selectedDate</code> with <code>v-model</code> from the
				consumer
			</td>
			<td>Mutate the events array directly — pass a new array</td>
		</tr>
		<tr>
			<td>
				Listen to <code>event-hover</code> and <code>event-leave</code> to position custom popovers
				using the provided <code>rect</code>
			</td>
			<td>Render the popover inside the event itself — the calendar clips overflow</td>
		</tr>
		<tr>
			<td>
				Provide stable <code>id</code>s on events so overlap layout and selection are consistent
				across renders
			</td>
			<td>Use array indices as event ids</td>
		</tr>
	</tbody>
</table>
