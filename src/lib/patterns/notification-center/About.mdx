# Notification Center

A bell-trigger popover that lists notifications. Two variants: a compact `simple` list, and a `complex` panel with tabs and infinite scroll. Built on top of Reka UI Popover, so the panel renders in a portal and respects collision detection.

## When to use

- App headers — the canonical "bell with red dot" entry point for inbound activity.
- A queue of action requests (approve/reject leave, accept shift offers, review swap requests) where each item carries inline buttons.
- A surface where users skim recent activity without leaving the page.

## When not to use

- For toast-style transient feedback — use the [Toast](?path=/docs/general-toast--docs) primitive.
- For a dedicated full-page inbox — link to the inbox route from the footer instead of cramming all data into the popover.
- For modal-blocking critical alerts — use a [Dialog](?path=/docs/general-dialog--docs).

## With actions

Items may carry an `actions` array — each action becomes a button row at the bottom of the item. Action click emits `action-click` with the item and the action key, so consumers wire approve/reject/etc. directly from store actions.

<Canvas of={NotificationCenterStories.SimpleWithActions} />

## High priority

Set `accent: true` on an item to render a left destructive border. Use sparingly for items that genuinely need attention.

## Complex with tabs and infinite scroll

<Canvas of={NotificationCenterStories.Complex} />

The list is virtualised — large unread queues stay smooth on low-end devices. `load-more` fires when the user scrolls near the bottom; consumers fetch the next page and append to `items`.
