# Sidebar

A composable sidebar navigation surface with collapsible states, mobile sheet behaviour, persisted open/closed state, and an icon-only rail mode. Built on a `SidebarProvider` context so any descendant can read the current state without prop drilling.

## When to use

- The primary navigation chrome of an app — left or right rail of the layout.
- Multi-section navigation where the user benefits from collapsing the rail to icons to reclaim horizontal space.
- Surfaces with grouped menus (Favorites, Records, etc.) that expand and collapse independently of the sidebar's own state.

## When not to use

- Simple top-of-page navigation — use a horizontal nav or breadcrumb instead.
- One-shot drawers (settings, filters that don't persist) — use `Drawer` or `Dialog`.
- Very narrow surfaces (< 320 px) where neither the icon rail nor the expanded sidebar fits — render a `Sheet`-style mobile menu via `SidebarTrigger` only.

## Basic usage

```vue
<SidebarProvider collapsible="icon">
  <Sidebar>
    <SidebarHeader>
      <!-- Logo, org switcher -->
    </SidebarHeader>

    <SidebarContent>
      <SidebarGroup>
        <SidebarGroupLabel>Navigation</SidebarGroupLabel>
        <SidebarGroupContent>
          <SidebarMenu>
            <SidebarMenuItem>
              <SidebarMenuButton is-active tooltip="Dashboard">
                <Home />
                <span>Dashboard</span>
              </SidebarMenuButton>
            </SidebarMenuItem>
          </SidebarMenu>
        </SidebarGroupContent>
      </SidebarGroup>
    </SidebarContent>

    <SidebarFooter>
      <!-- User info -->
    </SidebarFooter>
  </Sidebar>

  <main>
    <SidebarTrigger />
    <!-- Page content -->
  </main>
</SidebarProvider>
```

## Collapsed-state slots

Use `SidebarCollapsibleHide` and `SidebarCollapsibleShow` when expanded and collapsed states need different markup — e.g. a labelled `+` button when expanded vs. a centred 3-dots dropdown when collapsed.

```vue
<SidebarCollapsibleHide>
  <!-- Inline `+` button next to the Favorites label -->
</SidebarCollapsibleHide>

<SidebarCollapsibleShow>
  <!-- 3-dots dropdown centred in the icon rail -->
</SidebarCollapsibleShow>
```

## Composable

```ts
import { useSidebar } from '@fromforgesoftware/vue-kit';

const {
	state, // 'expanded' | 'collapsed'
	open,
	setOpen,
	toggleSidebar,
	isMobile,
	openMobile,
	setOpenMobile,
	collapsible, // 'offcanvas' | 'icon' | 'none'
	side, // 'left' | 'right'
} = useSidebar();
```

## Do / Don't

<table>
	<thead>
		<tr>
			<th>Do</th>
			<th>Don't</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>
				Wrap every <code>SidebarMenuButton</code> in a <code>SidebarMenuItem</code> so the rail's
				keyboard order and tree-branch styling stay correct.
			</td>
			<td>
				Render bare <code>SidebarMenuButton</code>s under <code>SidebarMenu</code> — the{' '}
				<code>&lt;ul&gt;</code> / <code>&lt;li&gt;</code> structure breaks.
			</td>
		</tr>
		<tr>
			<td>
				Provide a <code>tooltip</code> on every <code>SidebarMenuButton</code> so labels are still
				readable in <code>collapsible="icon"</code> mode.
			</td>
			<td>
				Rely on the inline <code>&lt;span&gt;</code> label alone — it's hidden when the rail
				collapses.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>SidebarCollapsibleHide</code> / <code>SidebarCollapsibleShow</code> to swap
				layouts between expanded and icon-collapsed states.
			</td>
			<td>
				Toggle visibility with <code>v-if="state === 'collapsed'"</code> — it bypasses the provider
				and won't react to mobile breakpoint changes.
			</td>
		</tr>
		<tr>
			<td>
				Use <code>SidebarMenuCollapsible</code> for nested sections — it auto-degrades to a dropdown
				when the rail is icon-collapsed.
			</td>
			<td>
				Hand-roll a nested <code>Collapsible</code> inside the icon rail — submenus don't fit and
				won't be reachable by keyboard.
			</td>
		</tr>
		<tr>
			<td>
				Persist open state via the provider's built-in <code>localStorage</code> handling.
			</td>
			<td>
				Re-implement persistence on top of <code>:open</code> / <code>@update:open</code> — you'll
				fight the provider on hydration.
			</td>
		</tr>
	</tbody>
</table>
