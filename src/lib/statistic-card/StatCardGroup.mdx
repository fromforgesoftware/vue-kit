# StatCardGroup

Layout container for a row or grid of <a href="?path=/docs/statistic-card-statcard--docs">StatCard</a>, <a href="?path=/docs/statistic-card-iconstatcard--docs">IconStatCard</a>, or <a href="?path=/docs/statistic-card-progressstatcard--docs">ProgressStatCard</a> children. Defaults to a connected surface with internal dividers — switch to `separated` to keep each card distinct.

## When to use

- A summary KPI row at the top of a dashboard or report.
- Any group of related metrics that share the same period.
- Wherever you'd otherwise hand-roll a `grid grid-cols-N` for stat cards.

## When not to use

- For two unrelated metrics — render them separately.
- For arbitrary card layouts — use the underlying CSS grid directly.

## Anatomy

<Canvas of={StatCardGroupStories.Default} />

A `<section role="group">` that lays out children in a CSS grid. The component injects classes that override the children's borders/radius in `connected` mode so the group reads as one card.

## Gap (separated layout only)

<Canvas of={StatCardGroupStories.SeparatedGaps} />

`none` (0), `sm` (8 px), `default` (16 px), `lg` (24 px). Connected layouts ignore gap — dividers handle separation.

## Composition

```vue
<StatCardGroup :columns="3">
  <StatCard label="Total Sales" value="$956.82k">
    <template #trend><StatTrend :value="5.4" direction="up" /></template>
    <template #footer>+8.20k vs prev. 60 days</template>
  </StatCard>
  <StatCard label="New Customers" value="1,245">…</StatCard>
  <StatCard label="Churn Rate" value="2.8%">…</StatCard>
</StatCardGroup>
```

The group accepts any combination of the four card variants — the override CSS targets `data-slot="stat-card"`, `icon-stat-card`, and `progress-stat-card` so all of them render flush in `connected` mode.

## Accessibility (WCAG 2.2 AA)

- **1.3.1 Info and Relationships**: rendered as `<section role="group">` so assistive tech announces it as a related cluster.
- **1.4.11 Non-text Contrast**: the divider colour (`border-border`) meets 3 : 1 against the card background.
- **Children own their accessibility**: each StatCard child carries its own `aria-label` and (when interactive) `role="button"` / `tabindex="0"`.

## Responsive behaviour

<Canvas of={StatCardGroupStories.InteractiveResponsive} />

The grid collapses to a single column on the smallest viewports, then expands at the breakpoints listed above.

		<tr>
			<td>Mix StatCard / IconStatCard / ProgressStatCard inside one group when meaningful</td>
